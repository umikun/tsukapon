**対象システム**: サイボウズOffice 未読メッセージ監視 → Chatwork 通知スクリプト
**稼働環境**: 社内Windows機（24時間稼働）
**実行方式**: Windowsタスクスケジューラ + Node.js + Playwright

---

## 1. 停止条件（いつ運用を停止するか）

### A. 即時停止すべき条件（緊急停止）

| # | 条件 | 検知方法 |
|---|---|---|
| A1 | APIトークンの漏洩・流出が疑われる | GitHubへの誤コミット、端末紛失、第三者アクセス痕跡 |
| A2 | 意図しない宛先・内容のメッセージ送信を検知 | ログ監視、誤通知の報告 |
| A3 | Chatwork APIレート制限超過の連続発生（HTTP 429） | エラーログに429が一定回数以上 |
| A4 | サイボウズOffice側の認証エラー連続発生（ログイン失敗） | 連続3回以上の認証失敗 |
| A5 | 情報セキュリティインシデント発生時（社内の他システム含む） | 情報システム部門からの指示 |
| A6 | Chatworkアカウント・サイボウズアカウントの停止/変更 | アカウント管理部門からの通知 |

### B. 計画停止条件（予定されたもの）

| # | 条件 | 対応 |
|---|---|---|
| B1 | サイボウズOfficeのメンテナンス・仕様変更 | 事前停止 → 動作確認後に再開 |
| B2 | Chatwork側のメンテナンス・API仕様変更 | 事前停止 → 動作確認後に再開 |
| B3 | Windows機のOS更新・再起動 | タスクスケジューラで自動復旧 |
| B4 | 利用者（本人）の退職・休職・異動 | 恒久停止 |
| B5 | 社内規程・セキュリティポリシーの変更 | 管理者判断により停止 |

### C. 品質劣化による停止条件

| # | 条件 | 閾値 |
|---|---|---|
| C1 | 連続エラー発生 | 5回連続で異常終了 |
| C2 | 通知の重複・欠落 | 1日に1件でも発生 |
| C3 | HTML構造変更によるスクレイピング失敗 | 1回でも発生次第、手動停止 |

## 2. 停止手順（どう止めるか）

### 手順①：スクリプト実行の即時停止（最優先・数秒で完了）

**Windowsタスクスケジューラからタスクを無効化する**

1. Windows機にログイン
2. 「タスクスケジューラ」を起動
3. 該当タスク（例：`CybozuWatcher`）を右クリック → **「無効」** を選択
4. 現在実行中のプロセスがあれば「終了」を選択

PowerShellで一括停止する場合:

```powershell
Disable-ScheduledTask -TaskName "CybozuWatcher"
Get-Process node -ErrorAction SilentlyContinue | Stop-Process -Force
```

### 手順②：Chatwork APIトークンの無効化（漏洩時・強い停止）

**トークンを無効化すれば、スクリプトが動いてもAPI呼び出しは100%失敗する。**

1. Chatworkにログイン
2. 「サービス連携」→「API Token」
3. 該当トークンを **「削除」または「再発行」**
4. 再発行した場合、旧トークンは即時無効化される

### 手順③：サイボウズ側セッションの遮断（必要時）

1. サイボウズOfficeにログイン
2. 「個人設定」→「パスワード変更」
3. パスワード変更によりスクリプトのログインを遮断

### 手順④：恒久停止（アンインストール）

1. 手順①②③を実施
2. `C:\cybozu-watcher\` ディレクトリ一式を削除
3. `.env` に記載の認証情報を破棄（ファイル上書き削除）
4. Node.js / Playwright が他用途で不要なら併せてアンインストール
5. 管理者へ停止完了を報告

---

## 3. 停止判断・実行の責任者

| 役割 | 担当 |
|---|---|
| 停止判断者 | 利用者本人（条件 A1〜A4、B3、C1〜C3） |
| 上位判断者 | 情報システム管理者（条件 A5、A6、B5） |
| 実行者 | 利用者本人 |
| 報告先 | 情報システム管理者・直属上長 |

---

## 4. 停止後の再開条件

- 停止原因の特定と恒久対策の完了
- 手動テスト実行による正常動作確認
- 管理者への再開報告（条件 A・B5 で停止した場合は承認必須）

---

## 5. 監査・記録

- すべての停止・再開をログファイル（`logs\YYYY-MM-DD.log`）に記録
- 緊急停止の場合は、管理者宛に **停止日時・原因・対応内容** をメールで報告

---

## 申請書記載用・要約文（短縮版）

> **停止条件**: ①APIトークン漏洩時、②誤送信検知時、③連続エラー5回以上、④セキュリティインシデント発生時、⑤仕様変更・メンテナンス時、⑥利用者の異動・退職時。
>
> **停止手順**: ①Windowsタスクスケジューラで該当タスクを無効化し実行プロセスを終了、②必要に応じてChatwork APIトークンを削除・再発行、③恒久停止時は関連ファイル・認証情報を削除のうえ管理者へ報告。
