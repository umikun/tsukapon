### 1.2 主要人物の詳細分析

#### Pieter Levels (@levelsio) — Build in Publicの教父

**コンテンツ構成**（一次観察）：
- **収益マイルストーンのツイート**：新しいMRRに達するたびにStripeのスクリーンショットを投稿。例：「$10K MRR after 3 weeks with 318 customers」→ 大量リツイート
- **技術的意思決定の実況**：新モデル（Fluxなど）の試用・A/Bテスト結果・ランディングページのコンバージョン率（1% → 4%）
- **失敗の透明化**：プロジェクトの97%が失敗したことを公開
- **プロジェクト間の戦略複製**：プロジェクト間でのcopy-paste戦略を公開共有

**主要データ**：
- 現在の月収 ~$138K/month（2025年11月データ）
- PhotoAIが収益の70%を占める（$106K/m）、その他InteriorAI・RemoteOKなど
- TikTok1本でPhotoAIの$7,000 MRR/日増加を実現

**戦略の本質**：Build in Publicとは「進捗を共有する」ことではなく、「オーディエンスをステークホルダーにする」こと。オーディエンスが0から$100K MRRへの成長を見守ることで投資家的な心理が生まれ、あなたの成功を望むために自発的に拡散してくれる。

> 出典：[FastSaaS: How Pieter Levels Built a $3M/Year Business](https://www.fast-saas.com/blog/pieter-levels-success-story/) + [X: @levelsio PhotoAI $10K MRR](https://x.com/levelsio/status/1631715500010135552) + [X: @levelsio PhotoAI $150K/mo](https://x.com/levelsio/status/1850305637303160853)

#### swyx (@swyx) — Learn in Public + Pick Up What They Put Down

**コア理念**：
1. **Learn in Public**：こっそり学んで潜伏しないこと。ブログを書く・チュートリアルを作る・フォーラムで質問と回答をする・YouTubeをやる——「学習の排気ガス」（learning exhaust）を作り出す
2. **Pick Up What They Put Down**：業界の有名人が新しいものを出したが、フィードバックが少ない。あなたがレビュー/解説/チュートリアルを書いてタグ付けする——「他人が自分の仕事を褒めてくれるなら一日中リツイートできる」ので彼らがリツイートしてくれる
3. **Macro-tweeting**：自分の古いツイート、特に「予測が当たった」ものを定期的に掘り起こす

**実際の運用**：
- 毎日AIニュースレター（Latent Space）を更新し、Twitterは彼の「公開ノート」として機能
- ツイートは公開ノートのため、ニュースレターは検索可能なAIニュースデータベースのため、図解は概念説明のため——**オーディエンスへの貢献は副産品**
- 「AI Engineer」というロールの定義を生み出した

**花叔（Huashu）への示唆**：swyxの戦略は、深みはあるが独自研究者ではない人に特に適している。核心は：**新しいものを発明する必要はなく、他人が発明したものを分かりやすく説明して原作者をタグ付けすればいい**。

## 二、X/Twitter 2026アルゴリズム機構（AI/テック分野で必ず知っておくべきこと）

### 2.1 三段階ランキングパイプライン

1. **候補フィルタリング**：毎日5億件のツイートから各ユーザー向けに~1,500件の候補を絞り込む（フォロー内50%・フォロー外50%）
2. **機械学習ランキング**：ニューラルネットワークが数千の特徴を分析し、10個の確率ラベルを出力
3. **Grok駆動の更新**（2026年1月）：transformerモデルが各投稿と動画を読み込み、毎日50億回のランキング決定を行う

### 2.2 シグナル重み付け公式

| インタラクションの種類 | 重み | いいねとの比較倍数 |
|---------|------|-------------------|
| いいね | x1 | 1x |
| ブックマーク | x10 | 10x |
| リンククリック | x11 | 11x |
| プロフィールクリック | x12 | 12x |
| リプライ | x13.5 | 13.5x |
| リツイート | x20 | 20x |
| **会話（リプライ＋作者の返信）** | **x75** | **150x** |

**重要な洞察**：質の高い会話1回 = いいね150個分のアルゴリズム上の価値。これがAI/tech KOLが積極的にコメントに返信する理由を説明している。

### 2.3 AI/テック分野特有のアルゴリズムポイント

**エンゲージメント速度（Engagement Velocity）が最強のシグナル**：
- 最初の15〜30分のインタラクションがすべてを決める
- 15分以内に10+インタラクション → 指数関数的な拡散
- 15分以内に3件未満 → ツイートは死亡
- **対策**：オーディエンスが最も活動的な時間帯に投稿する（AI/tech グローバルオーディエンス向け：Pacific Time 8-10 AM、日本時間で深夜0時〜2時）

**時間減衰**：6時間ごとに可視性が半減。AIニュースは時効性があるため、迅速な対応が重要。

**外部リンクのペナルティ**：
- リンク付きツイートのリーチが30〜50%低下（非Premiumユーザーはほぼゼロのエンゲージメント）
- **解決策**：メインツイートにリンクを入れず、最初のリプライにリンクを掲載
- 2026年3月以降、Premiumユーザーのリンクペナルティはほぼ解消

**X Premiumの効果**：有料ユーザーは2〜4倍のリーチ効果を得る。Xを真剣にやる人にとっては必要な投資。

## 三、AI/テック分野特有のコンテンツ戦略

### 3.1 コンテンツタイプとエフェクトマトリクス

| コンテンツタイプ | エンゲージメント | 頻度の推奨 | 例 |
|---------|--------|---------|------|
| **新モデル/プロダクト速評** | 非常に高い | ホットな話題があれば投稿 | "GPT-5.3発売、3つのシナリオでテストした..." |
| **Build in Public更新** | 高い | 週2〜3回 | MRRのスクリーンショット・機能リリース・ユーザーフィードバック |
| **技術チュートリアル/スレッド** | 高い | 週1回 | 8〜12ツイートのチュートリアルスレッド |
| **デモ動画/GIF** | 高い | 成果があれば投稿 | 15〜30秒のプロダクトデモ |
| **Hot Take/議論を呼ぶ意見** | 中〜高 | 慎重に使う | "Vibe coding is a slur" |
| **論文解説スレッド** | 中 | 週1回 | 簡単な言葉で重要な発見を解説 |
| **ツール比較/レビュー** | 中 | 月2〜3回 | スクリーンショット＋テスト結果の表 |
| **個人的なストーリー/感想** | 中 | 時々挟む | 起業の心路歴程・転身の経験 |
| **ミーム/ユーモア** | 変動が大きい | 慎重に | AI関連のネタ画像 |

### 3.2 新モデル発表：迅速な対応戦略

AIセクター最大のチャンスウィンドウは**新モデルの発表**（GPT-5・Claude Opus・DeepSeekなど）。これは他のテック分野と差別化するコアな特徴。

**対応タイムライン**：
1. **発表後0〜1時間**：Quick Take（最初の反応 + 明確な意見）を投稿
2. **発表後1〜6時間**：デモ/テスト結果（スクリーンショット + GIF）を投稿
3. **発表後6〜24時間**：深いスレッド（体系的テスト + 比較 + 意見）を投稿
4. **発表後1〜7日**：深い記事/動画（完全レビュー + 実践ケース）を投稿

**OpenAIのやり方**（参考になる）：Sam Altmanは発表から数分以内に「あなたたちはこれで何をしたいですか？」とツイートし、コミュニティ自身にコンテンツを生み出させた——一方的な宣伝ではなく。

> 出典：[FutureSocial: How OpenAI Used Twitter Replies to Create Launch Content](https://futuresocial.beehiiv.com/p/openai-used-twitter-replies-create-launch-content) + 一次観察

### 3.3 Build in Publicの具体的な操作マニュアル

**共有すること**：
- MRRのマイルストーン + Stripeのスクリーンショット（[BrandBird MRR Meter](https://www.brandbird.app/tools/twitter-mrr-meter)で標準化した画像を生成）
- 機能リリース + デモスクリーンショット/動画
- 失敗の振り返り（post-mortem）
- 技術スタックの選択と意思決定の理由
- ユーザーフィードバックのスクリーンショット
- 月次/四半期まとめスレッド

**共有しないこと**：
- 正確な顧客獲得コスト（CAC）とユニットエコノミクス（競合上の機密）
- 顧客の個人情報
- コア競争優位性の具体的な実装の詳細

**フォーマットのコツ**：
- スレッドの冒頭にフックを使う：「Week 12 of building [Product]: Hit $2K MRR...」
- スレッドの末尾にCTAを使う：「Follow along for weekly updates」
- ビジュアルコンテンツは5倍多いエンゲージメントを獲得
- 各リプライに1時間以内に返信する

**ケースデータ**：
- AudioPen：12時間で開発 → 2日で100人の有料ユーザー → Product Hunt #1 → 最初の2ヶ月で$73Kの収益
- SiteGPT：Twitter 24K+フォロワー → Product Hunt #1 → 6ヶ月で$15K MRR → $95K MRR
- あるインディーハッカー：4ヶ月でTwitterフォロワーが2,400人に成長 → プロダクトリリース時に$8K MRR

> 出典：[OpenTweet: Build in Public Guide](https://opentweet.io/blog/build-in-public-twitter-guide-saas-founders) + [Teract: Twitter Strategy for Indie Hackers 2026](https://www.teract.ai/resources/twitter-strategy-indie-hackers-2026) + [AudioPen Starter Story](https://www.starterstory.com/stories/audiopen) + [SiteGPT Rise to $15K MRR](https://www.indiehackers.com/post/from-side-hustle-to-ai-star-sitegpts-rise-to-15k-mrr-ff15fee186)

### 3.4 スレッド執筆のベストプラクティス

**データの裏付け**：8〜12ツイートのスレッドは短いスレッドより47%高いパフォーマンスを示す（Sprout Social 2026データ）。スレッド全体は単独ツイートよりも3〜5倍多いエンゲージメントを獲得。

**構造テンプレート**（AI/tech向け）：

```
ツイート1（フック）：驚くべきデータ/直感に反する意見 +「スレッド」
ツイート2-3：背景と問題の定義
ツイート4-8：核心的な論拠/ステップ/発見
ツイート9-10：実際の操作/コード/スクリーンショット
ツイート11：まとめ + 重要な気づき
ツイート12：CTA（フォロー/ブックマーク/リツイートのお願い）
```

**AI分野特有のスレッドタイプ**：
1. **「Xをテストした、結果は驚くべきものだった」**型：新モデル/ツールの実測スレッド
2. **「0から$XK MRRへのN個の教訓」**型：Build in Publicのまとめ
3. **「この論文が私の認識を変えた」**型：論文解説
4. **「X vs Y：深い比較」**型：ツール/モデルの横断評価
5. **「AIでXを作り、N時間を節約した」**型：実践ケース

## 四、ビジュアルコンテンツ戦略（コードスクリーンショット・GIF・デモ動画）

### 4.1 各コンテンツフォーマットの効果比較

| フォーマット | エンゲージメント率 | 最適時間/サイズ | 適用シーン |
|------|--------|-------------|---------|
| テキストのみ | 0.1% | 120〜130文字が最適 | 意見・hot take |
| 画像/スクリーンショット | 0.08% | 16:9横型 | コードスクリーンショット・データ表 |
| GIF | 中程度 | 3〜8秒ループ | 機能デモ・インタラクション効果 |
| 動画 | 0.42% | 15〜30秒 | プロダクトデモ・チュートリアル |
| スレッド | 単独ツイートの3〜5倍 | 8〜12ツイート | 深いコンテンツ・チュートリアル・レビュー |

**注意**：Xはテキストが動画に引けを取らない唯一の主要プラットフォーム。しかし動画の0.42%のエンゲージメント率は画像の0.08%を大きく上回る。

### 4.2 コードスクリーンショットのツールとテクニック

- **[Snappify](https://snappify.com/)**：アバターとユーザー名を追加した美しいコード表示画像を作成
- **[Pika](https://pika.style/templates/code-image)**：複数のテーマに対応したコードスクリーンショットを生成
- **[Codeshot](https://codeshotapp.com/)**：テーマを選択してTwitterサイズでエクスポート

**重要な原則**：
- コードスクリーンショットは重要な行を強調し、ページ全体のコードを貼らない
- コメント/ハイライトで重要なポイントに印をつける
- 最初のフレームをビルボードとして扱う——太字・高コントラスト・明確な約束

### 4.3 デモ動画のベストプラクティス

- **16:9横型**がデモと画面録画に最適
- **15〜30秒**が最適な時間（視聴完了率を最大化）
- **オーディエンスが音声オフで視聴することを前提に**：重要な情報は字幕で表現
- **最初のフレームがカバーになる**：フィードの中でビルボードの役割を果たす
- **メイン動画を投稿後、リプライスレッド**で要点・タイムスタンプ・リンクを補足

> 出典：[ScriptStorm: Twitter Video Best Practices](https://scriptstorm.ai/blog/twitter-video-best-practices-length-format-engagement) + [Snappify](https://snappify.com/) + [Codeshotapp](https://codeshotapp.com/posts/how-to-share-code-on-twitter/)

## 五、オープンソースプロジェクトの普及戦略

### 5.1 Twitter/Xでの普及のための重要な操作

1. **GitHub Social Preview**：リポジトリの設定で美しいプロモーション画像をアップロードし、共有リンクをより目立たせる（多くのプロジェクトがこれを忘れている）
2. **継続的な発信**：主な戦略はkeep yapping——小さなアップデート・コーディングの旅・技術的意思決定を発信し続ける
3. **Listicle相互タグ戦略**：同カテゴリのプロジェクトを含む一覧記事を書き、Twitterで各メンテナーをタグ付けする——「自分の仕事を褒められると一日中リツイートできる」ので彼らがいいね/リツイートしてくれる
4. **Awesomeリスト**：GitHubのawesome-xxxリストへPRを提出する
5. **マルチプラットフォーム展開**：火曜〜木曜のPacific Time 8-10 AMに発表し、各プラットフォームに合わせてコピーを調整する

**コア発見**：ツイートは新しいスターと新しい貢献者の獲得に有意なプラスの効果をもたらす。アクティブなTwitterコミュニティは新しい貢献者を引き付ける上で重要な役割を果たす（学術論文による検証）。

### 5.2 バイラルな副産品戦略：ExaのTwitter Wrapped

**ケース**：Exa（AI検索エンジン）が「Twitter Wrapped」ツールで170万ユーザーを獲得。

**やり方**：
- 12月26日に公開：AIがユーザーのXアカウントを分析し、パーソナライズされた年間まとめ・突っ込み・未来予測を生成
- 4時間以内に50万ビュー
- 4日後：59,000リツイート・1,360万ビュー

**なぜ成功したか**：Spotify Wrappedと同じ原理——**天然に共有しやすいパーソナライズされたコンテンツ**。ユーザーが自分の結果を共有 → 友人が気になる → 自分も生成しに行く → 循環拡散。

**示唆**：AIプロダクトは**無料で・パーソナライズされた・共有可能な副産品**を作ることでバイラルな拡散を獲得できる。プロダクト自体をバイラルにする必要はなく、バイラルな入口が必要。

> 出典：[Indie Hackers: Exa Twitter Wrapped](https://www.indiehackers.com/post/tech/exa-an-ai-powered-search-engine-gains-1-7m-users-with-viral-twitter-wrapped-vUAEDrWM4ELz5UHcbyjG) + [DEV: Promoted Open Source Repo to 6K Stars](https://dev.to/wasp/how-i-promoted-my-open-source-repo-to-6k-stars-in-6-months-3li9) + [FreeCodeCamp: 4.5K Stars in 6 Months](https://www.freecodecamp.org/news/how-to-get-more-engagement-with-your-open-source-project/) + [arXiv: Impact of Twitter Mentions on GitHub](https://arxiv.org/html/2401.02755)

### 6.1 成功事例

**Han Xiao (@hanaborxiao) — Jina AI創業者**：

### 6.2 中国人開発者特有の課題と戦略

1. **言語の壁**：英語での執筆は越えなければならないハードルだが、完璧である必要はない——AIセクターは非ネイティブスピーカーに対してより寛容

## 七、AI/テック分野のトピック分類とマネタイズへの道筋

### 7.1 十大トピックタイプ（エンゲージメント順）

1. **新モデル/新機能の速評**：いち早くテストして意見を出す（エンゲージメント最高、時効ウィンドウ最短）
2. **Build in Publicのマイルストーン**：MRRのスクリーンショット・ユーザー数の突破（高エンゲージメント＋高信頼構築）
3. **実践チュートリアルスレッド**：「XでYをする方法」（高保存率、長尾トラフィックが良い）
4. **ツール比較横断評価**：「Claude vs GPT vs GeminiのXシナリオでのパフォーマンス」（高い検索価値）
5. **Hot Take/議論を呼ぶ意見**：「Vibe coding is a slur」（高い議論、リスクあり）
6. **個人的な失敗/教訓**：「Xをやって、Yの損失を出した」（高い共感、真実性の構築）
7. **論文解説**：簡単な言葉で解体（中程度のエンゲージメント、高い専門性シグナル）
8. **リソースまとめ**：「最高のXツール10選」（高保存率）
9. **業界トレンド予測**：「2026年AIの5つのトレンド」（変動が大きく、当たれば高リターン）
10. **ミーム/ユーモアコンテンツ**：AI関連のネタ（低ハードルの拡散だが、専門性は構築しない）

### 7.2 コンテンツからマネタイズへの道筋

```
Xツイート/スレッド → 個人ブランドの認知
    |
ブログ/ニュースレター（深いコンテンツ）→ メーリングリスト
    |
Product Hunt/GitHubローンチ → ユーザー獲得
    |
有料プロダクト/コンサルティング/コース → 収益
```

**重要なポイント**：X上のコンテンツは直接マネタイズにつながらず、信頼とオーディエンスを構築する。マネタイズは深いコンテンツ（ニュースレター・ブログ）とプロダクトのローンチ段階で発生する。

## 八、戦術クイックリファレンスカード

### 8.1 投稿リズム

| コンテンツタイプ | 頻度 | 時間 |
|---------|------|------|
| 日常ツイート（意見・小さなアップデート） | 毎日3〜5件 | 2〜3時間間隔 |
| スレッド（深いコンテンツ） | 週1〜2回 | 火曜〜木曜 |
| 他者へのリプライ | 投稿量の70% | 終日 |
| 新モデル速評 | あれば即投稿 | リリース後1時間以内 |

### 8.2 成長の公式

**0〜1Kフォロワー段階**：
- 70%の労力をリプライに、30%を投稿に
- 業界の大手アカウントのツイートに返信し、有益な補足を提供する
- swyxのPUWTPD戦略：有名人の新作品にレビュー/チュートリアルを書く

**1K〜10Kフォロワー段階**：
- コンテンツの柱を確立（3〜5つの固定テーマ）
- 週1〜2本のスレッドで専門性を構築
- Build in Publicを開始

**10K+フォロワー段階**：
- ニュースレター/ブログで深いコンテンツ資産を構築
- プロダクトローンチに既存オーディエンスを活用
- 選択的なコラボプロモーションを開始

### 8.3 AI分野特有の成長ハック

1. **新モデルのリリース日がスーパーボウルだ**：全員がAIニュースをチェックしており、関連コンテンツは自然にトラフィックを集める
2. **無料ツール = 顧客獲得の入口**：ExaのTwitter Wrapped・Pieterの様々な無料AI toy
3. **オープンソース = 信頼の加速器**：オープンソースプロジェクトはX上でクローズドソースプロダクトよりはるかに高い信頼を獲得
4. **スクリーンショット > 説明**：常に視覚的な証拠を使う（Stripeのスクリーンショット・プロダクトデモ・コードの結果）
5. **スレッドは長文コンテンツの武器**：X上のスレッドは他のプラットフォームのブログ記事と同等
6. **リプライは最も過小評価されている成長レバー**：良いリプライ1件のアルゴリズム上の重み = いいね13.5個

## 九、一般的なTwitter戦略とAI/テック分野の特性の違い

| 次元 | 一般的なTwitter | AI/テック分野 |
|------|------------|------------|
| **時効性** | 事前にスケジュールできる | 新モデルのリリースには時間単位の対応が必要 |
| **コンテンツの深さ** | 短くて簡単なものが中心 | スレッドと技術解説がコア資産 |
| **ビジュアルコンテンツ** | 美しい画像・インフォグラフィック | コードスクリーンショット・ターミナル録画・デモGIF |
| **信頼の構築** | 個人ブランドストーリー | オープンソース貢献＋技術的深さ＋収益の透明性 |
| **オーディエンスの特性** | 幅広い消費者 | 開発者/起業家（価値が高いが騙しにくい） |
| **リンク戦略** | できるだけ避ける | 必ず共有する必要がある（GitHub/Blog）が、リプライに入れる |
| **成長の道筋** | フォロワー → ブランドコラボ | フォロワー → プロダクトユーザー/オープンソース貢献者 |
| **国際性** | ローカライズが顕著 | AIコミュニティは自然にグローバルで、英語が共通語 |
| **検証基準** | フォロワー数/エンゲージメント数 | 実際に何かを作れるか（ship or shut up） |

## 十、花叔（Huashu）のX戦略への具体的な提言

以上の調査を踏まえ、花叔（Huashu）の属性（AI Native Coder・インディー開発者・30万+のSNSフォロワー）と組み合わせて：

1. **明確なポジショニング**：「中国人インディー開発者がAIでプロダクトを作る」——このアイデンティティは英語のX上で独自の価値を持つ（中国AIエコシステムの一次情報 + インディー開発者の物語）
2. **コンテンツの柱の提案**：Build in Public（プロダクトデータ）＋ AIツール実測 ＋ 中国AIの視点
3. **迅速な対応**：新モデルのリリース時に、中国人開発者の視点で速評を行う（差別化）
4. **プロダクトをコンテンツに**：小猫補光灯・GLM Codeなどのプロダクトの開発ストーリーは自然にBuild in Publicに適している
5. **スレッド中心**：週次スレッドを発行し、日常的にはリプライ中心で、量を追わない
6. **視覚的な証拠**：プロダクト関連のツイートには必ずスクリーンショット/GIF/動画を添付
7. **バイリンガル分離**：Xは英語、公式アカウント/小紅書は中国語、混在させない
