# コア心智モデル（6つ）+ 意思決定ヒューリスティック（10条）

> 必要に応じて読み込む：ユーザーが「なぜこうするのか」「根本的なロジック」「思考フレームワーク」を聞いてきたとき、またはシナリオA/Bで深い説明が必要なときに参照する。

---

## コア心智モデル（6つ）

### モデル1: リーン検証フライホイール

**一言で**：まず最小限のコンテンツで検証し、有効なら拡張、拡張したら新コンテンツに還元する。

**仕組み**：
```
Tweet（アイデアを検証）
  ↓ データが良好？
Thread（深度を展開）
  ↓ データが良好？
Newsletter/Blog（長文資産）
  ↓ データが良好？
動画/コース/プロダクト（マネタイズ）
  ↑ 新アイデアで還元 ←──────┘
```

**出典**：Cole/Bush(Lean Writing)、Sahil(225+件のthreadはすべて先に検証)、Hormozi(tweet→video パイプライン)、Welsh(Content OS)——4つの流派が独立して同じパターンに収束。

**応用**：長いコンテンツを書く前に「このアイデアはツイートで検証したか？」と自問する。

**限界**：低頻度・高品質路線（Karpathyなど）はこのフライホイールに依存せず、個人の権威とコンテンツの希少性に頼る。フォロワーを構築中のステージに適しており、すでに数百万フォロワーを持つ権威者には向かない。

### モデル2: アテンション・エンジニアリング

**一言で**：各コンテンツの最初の2行が生死を決める。フックは工学的に設計できる。

**コア公式**：

フック品質 = 好奇ギャップ × 信頼性 × 具体性

- **好奇ギャップ**（Cole）：冒頭と結末を明かし、中間を隠す——読者にクリックを強制する
- **信頼性**：数字・人名・時間のアンカー（「1000アカウントを調査した結果...」）
- **具体性**：「不快なほど具体的」になるまで限定詞を加える（ColeのHeadline Checklist）

**HormoziのValue EquationをHookに適用**：
```
フック価値 = (期待する結果 × 信頼性) / (時間コスト × 努力量)
```
分子が大きく分母が小さいほど、フックは抗いがたくなる。

**アルゴリズム検証**：XのEngagement Velocity機構——最初の15〜30分のエンゲージメント速度が、ツイートがより大きなトラフィックプールに入るか死ぬかを決める。フックがこのウィンドウ期の結果を左右する。

**操作ルール**：
- フックに創作時間の50%を使う（10〜15バージョンを書いて最良を選ぶ——Cole）
- タイトルは三問に答える：誰に向けて？何を伝える？なぜ読む必要がある？
- `writing-workshop.md > フック改善例` でbefore/after比較を参照

**限界**：フックを過度に最適化するとクリックベイトになる。コンテンツはフックの約束を履行しなければならず、さもなければ長期的な信頼が損なわれる。

### モデル3: カテゴリー創造

**一言で**：既存の競争レーンに割り込むのではなく、自分だけのカテゴリーを創る。

**3段階の進化**：

| レベル | 戦略 | 例 |
|--------|------|----|
| 初級 | 細分化したニッチに絞る(niche down) | 「AIツールレビュー」 |
| 中級 | Interest Stack の組み合わせ | 「AI + 個人開発 + プロダクト思考」 |
| 上級 | カテゴリー創造(Category Creation) | 新語を発明し、レーンを再定義する |

**ColeのSnow Leopard理論**：「ライオン」（既存カテゴリーで誰が最強かを競う）になるのではなく、「ユキヒョウ」（希少な領域で独自のポジションを占める）になる。

**KoeのNiche of One**：
- ニッチを探すのではなく、ニッチを創る
- 公式：広いブランド（多様な興味をシェア）+ 絞り込んだプロダクト（具体的な問題に対応）
- Interest Stack：複数の興味を組み合わせて独自の視点を創る（フィットネス＋哲学＋ビジネス＋ライフスタイル）

**Languaging（命名術）**：自分の独自メソッドに専用の名前をつける。2語だけでカテゴリー認識を変えられる（car → electric car）。Ship 30 for 30自体がlanguagingの好例。

**Tequila Test（カテゴリー検証）**：
1. あなたのトピックに関する一般的なアドバイスをすべて書き出す
2. すべて消す
3. 残ったものを書く——消した後に何も書けなければ、まだ本当の差別化観点がない証拠

**応用**：ポジショニングが不明確なとき、他者と同質化を感じるとき、レーンが混みすぎていると感じるとき。

**限界**：カテゴリー創造には時間の蓄積と深い専門性が必要。コールドスタート段階では、まず既存カテゴリー内でフォロワーを積み上げる必要があるかもしれない。

### モデル4: 価値の先出し

**一言で**：秘密を無料で公開し、実行をセールスする。各コンテンツが一度の価値提供。

**Hormoziのコアインサイト**：自分でやる人は1%だけ、99%の人は誰かにやってもらうために費用を払う。高価値コンテンツを無料で提供 → 解決策があることを証明 → 信頼と互恵性の構築 → 自然なコンバージョン。

**三段式コンテンツ構造（Hook-Retain-Reward）**：
1. **Hook**：一瞬で注意を奪う（衝撃/疑問/大胆な約束）
2. **Retain**：継続的に価値を提供する（ストーリー＋オープンループ＋無駄ゼロ）
3. **Reward**：約束以上を届ける（実行可能なアクション提案、期待を超えて）

**Welshの教育優先**：まず教育コンテンツで権威を築き、その後マネタイズへ誘導する。18週間で44Kフォロワー増加のコアは「午後に教育系Tipsを投稿」。

**Sahilのファインマン式検証**：最もシンプルな言葉で複雑な概念を説明できないなら、まだ理解していない。文章はファインマンテクニックの公開実践。

**応用**：各コンテンツを書く前に「読者は読後すぐに何ができるか？」と問う。答えが「何もない」なら書き直す。

**限界**：純粋な価値提供だけでは個人的なつながりが生まれない。個人のストーリーと意見を織り交ぜる必要がある（Dickie Bushの75/25法則：75%は広い内容で新規獲得、25%は深い内容で定着促進）。

### モデル5: 公開制作

**一言で**：プロセスをコンテンツにし、オーディエンスをステークホルダーにする。

**2つのバリエーション**：

**Build in Public（levelsio）**：
- 収益（MRRのスクリーンショット）・プロセス（機能の反復）・失敗（97%のプロジェクトが失敗）を公開
- コアメカニズム：オーディエンスが$0から$100K MRRまでの過程を見守ることで「投資家マインド」が生まれる——彼らはあなたの成功を望み、自発的に拡散する
- シェアすること：MRRのマイルストーン・機能リリース・失敗の振り返り・技術スタックの意思決定・ユーザーフィードバック
- シェアしないこと：正確な顧客獲得コスト・顧客の個人情報・コア実装の詳細

**Learn in Public（swyx）**：
- 学習プロセスを公開：ブログを書き・チュートリアルを作り・フォーラムで質問と回答——「学習の排気ガス」を創る
- **Pick Up What They Put Down**：インフルエンサーが新しいものを出したときにレビュー/解説/チュートリアルを書いてタグ付けする——「自分の仕事を褒めてくれる投稿は一日中リツイートできる」から彼らは転載する
- 新しいものを発明する必要はない、他者が発明したものをわかりやすく説明するだけでいい

**応用**：AI/テクノロジーレーンのコア差別化戦略。個人開発者・プロダクトを作っているクリエイター・技術学習者に適している。

**限界**：実際に何かを作っていることが前提。純粋なコメンテーター型クリエイターはbuild in publicできない。心理的な強さも必要——公開とは失敗も公開することを意味する。

### モデル6: システム化された複利

**一言で**：インスピレーションをテンプレートとシステムで代替し、コンテンツ生産を予測可能な機械にする。

**WelshのContent OS**：
1. **キュレーション(Curate)**：インスピレーションとパフォーマンスの良いコンテンツを収集
2. **テンプレート化(Templatize)**：成功コンテンツの構造を抽象化してテンプレートに
3. **高速制作(Rapid Create)**：テンプレート＋素材を使い、1時間に10〜20件のコンテンツを生産
4. **配信(Distribute)**：クロスプラットフォーム＋自動化ツール

**Koeの2 Hour Writer**：
- 1時間の散歩でインスピレーション収集 ＋ 1時間で執筆・編集
- Idea Museum：整理された素材ライブラリ
- 執筆フレームワーク早見表：Listicle / 短投稿(個人の再定義/刺さる真実) / PSBストーリーアーク

**コンテンツ再利用フライホイール**：
```
Newsletter（週1〜2本の長文）
  ├── 5〜7件の短投稿を抽出 → Twitter/X
  ├── Twitterの投稿スクリーンショット → Instagram/LinkedIn
  ├── Newsletterの朗読 → YouTube
  └── 高エンゲージメントのツイート → 次回Newsletterのネタ選びに拡張
```

**SahilのNotionかんばん**：生の思いつき → 近日書く予定 → 執筆中 → 完成・未投稿 → 投稿済み。5列で管理し、コンテンツ切れなし。

**応用**：書けなくなったとき・ネタ枯れのとき・アウトプットが不安定なとき。システムがあれば調子の悪い日でも合格水準のコンテンツを出し続けられる。

**限界**：過度なシステム化でコンテンツが機械的になる。20〜30%の「システム外」スペースをインスピレーションと即興反応のために残しておく必要がある（特にAIレーンのトレンド対応）。

## 意思決定ヒューリスティック（10条）

### 1. ツイートを先に投稿してから長文を書く ← モデル1の応用
長いコンテンツを書きたい？まず1件のツイートで検証する。「Twitterはアイデアの精製所であり、放送チャンネルではない。」（Bush）
- **トリガー**：thread/newsletter/動画を書きたいと思ったとき
- **行動**：1件のツイートでコアな主張をテストし、データが良ければ拡張

### 2. フックに創作時間の50%を使う ← モデル2の応用
10〜15バージョンを書いて最良を選ぶ。タイトルは必ず答える：誰に向けて？何を伝える？なぜ読む必要がある？「フック改善例」を参照。
- **トリガー**：任意のコンテンツを書き始めるとき

### 3. 対話がすべてを圧倒する
アルゴリズムの重み：対話リプライ 150x > リプライ 27x > ブックマーク 20x > リツイート 2x > いいね 1x。作者からのリプライを引き出す対話1件は、いいね150件より価値がある。
- **トリガー**：エンゲージメント戦略を考えるとき
- **行動**：リプライを引き出すコンテンツを書く（質問・議論を呼ぶ意見・フィードバック依頼）、すべてのコメントに積極的に返信する

### 4. 1/3/1リズム
1文のフック ＋ 3文の展開 ＋ 1文のつなぎ。コンテンツをスキャン可能にする。単文行は「チェックポイント」として機能し、読者に小さなドーパミン刺激を与える。
- **トリガー**：3文を超える内容を書くとき
- **バリエーション**：1/4/1・1/5/1・1/2/5/2/1

### 5. スーパーボウル対応（AIレーン）
新モデルのリリース ＝ AIレーンのスーパーボウル。対応タイムライン：0〜1h クイックテイク → 1〜6h デモ → 6〜24h 深掘りThread → 1〜7日間 完全レビュー。
- **トリガー**：GPT/Claude/Gemini/オープンソースモデルの重大リリース
- **ポイント**：スピード > 完璧。最初に洞察を持つ声は、100番目の完璧な分析より価値がある

### 6. オーディエンスを所有する
アルゴリズムは変わるが、メールリストは変わらない。各ツイートの最終目的はnewsletterへの誘導。Twitterは発見エンジン（top of funnel）、newsletterは深い関係（owned audience）。
- **トリガー**：コンテンツ戦略を立てるとき
- **Sahilのデータ**：newsletter広告の月収$70K+、全額を成長に再投資

### 7. 4Aネタ選びマトリクス
1つのトピック × 4角度 = 無限のネタ選び：
- **Actionable**：やり方を教える（Tips/Guides/How-to）
- **Analytical**：データで裏付ける（Stats/Trends/Frameworks）
- **Aspirational**：可能性を鼓舞する（Lessons/Mistakes/Habits）
- **Anthropological**：人間本性に触れる（Fears/Failures/Lies/Struggles）
- **トリガー**：「書くことがない」と感じるとき

### 8. 秘密を公開して実行をセールスする ← モデル4の応用
「無料でシェアすべきか迷ったとき」の答えは公開する。99%の人は自分ではやらない。（Hormozi：広告費ゼロ・6ヶ月で100万フォロワー）
- **トリガー**：あるメソッド/ツール/プロセスを無料でシェアすべきか迷ったとき

### 9. テンプレートはインスピレーションより強い
成功したコンテンツを抽象化してテンプレートに変え、テンプレートで高速に生産する。Coleはわずか7種類のThreadテンプレートで200件以上のthreadを書いた。WelshはContent OSで1時間に10〜20件のコンテンツを生産。
- **トリガー**：効果の良いコンテンツを書いたとき、すぐにテンプレートを抽出する
- **Coleの7種類のテンプレート**：Framework / Story / Actionable / Curation / Lessons / Mistakes / Contrarian

### 10. コメント欄は金鉱
大手アカウントのコメント欄に高品質なリプライを残す ＝ トラフィックを借りる。Welsh：1件のリプライで6,700回の露出を獲得。Sahilのコールドスタート：threadを50の大手アカウントのコメント欄に投稿し、Chamathの1度のリツイートで成長が爆発。
- **トリガー**：フォロワー1万未満のコールドスタート段階
- **ルール**：「良い投稿ですね」は書かない。200〜400文字の「ミニニュースレター式リプライ」を書く
