## まず全体像を見てください

仕組みはシンプルです。

```
あなた（CEO）
　↓ やりたいことを1行書く
秘書（AIエージェント）
　↓ 内容を判断して各部署に振る
┌─────────────────────────┐
│ リサーチ部（バズ分析・ネタ発掘）      │
│ コンテンツ制作部（投稿・記事を生成）  │
│ エンゲージメント部（数字分析・改善案） │
│ マーケティング部（プロフ・導線の最適化）│
│ 商品開発部（商品企画・価格設計）      │
└─────────────────────────┘
　↓
完成品がフォルダに自動保存
```

ポイントは、
これが全部「1つのAIエージェント」の中で完結することです。

複数のツールを行き来する必要はありません。

フォルダの中に「ルールを書いたファイル」を置く。
AIがそのルールを読んで、部署として動く。

たったこれだけの仕組みです。

プログラミングは一切使いません。
日本語でルールを書くだけです。

では、各部署の具体的な作り方を公開していきますね。

## 秘書の作り方（ここが一番重要です）

6部署の中で、最初に作るべきは「秘書」です。

なぜかというと、
秘書がいないと指示のたびに
自分で「これはどの部署の仕事だっけ…」と
考えないといけなくなるからです。

秘書がいれば、1行投げるだけで全部やってくれます。

作り方はこうです。

フォルダの中に「secretary-rules.md」というファイルを作って、
以下を書いてください。

### secretary-rules.md

> あなたはこのAI組織の秘書です。
> 私から受けたタスクを分析して、
> 最適な処理を自動で実行してください。

**【振り分けルール】**

- 「投稿作って」「ポスト書いて」
  → brand-voice.md と target-dna.md を読んで投稿を生成
- 「伸びてる投稿を調べて」「競合を分析して」
  → 指定したアカウントやジャンルのバズ投稿を分析
- 「今週の数字を見て」「改善点を出して」
  → 投稿データを分析してTOP3/WORST3と改善案を出力
- 「プロフを見直して」「固定ポスト改善して」
  → フォロー率が上がるプロフ改善案を3パターン出力
- 「商品を考えて」「noteの企画作って」
  → ターゲットの悩みから商品コンセプトを設計
- 複合タスクの場合
  → 必要な処理を順番に実行（例：分析→その結果を元に投稿作成）

**【出力ルール】**

- 処理結果はファイルに保存すること
- 何をどう処理したか、最後に簡潔に報告すること

---

これだけで秘書が完成です。

使い方を見せますね。

**例①**：「来週のX投稿を10本作って。先週伸びたテーマ寄りで。」

→ 秘書が判断します
→ まず先週の投稿傾向を分析
→ 伸びたテーマを特定
→ そのテーマに沿って10本を生成
→ ファイルに保存

自分が書いたのは1行だけ。
裏では2つの処理が自動で連携しています。

**例②**：「競合の〇〇さんの直近投稿を分析して、強い型を見つけて、それを参考に投稿5本作って」

→ 秘書がリサーチ→分析→制作を一気に実行
→ 競合分析レポート＋投稿5本がまとめて出てきます

1行投げるだけで、3部署分の仕事が終わる。
これが秘書の威力です。

## リサーチ部の作り方（X運用の「目と耳」）

投稿が伸びない最大の原因、知ってますか。

「何を書くか」の精度が低いことです。

どんなに文章力が高くても、
ターゲットが興味ないテーマで書いたら伸びません。

リサーチ部は、
「今、何を書けば伸びるか」を教えてくれる部署です。

作り方はこうです。
秘書のルールに加えて、
リサーチ専用のルールファイルを追加します。

### research-rules.md

> あなたはリサーチ部の専門家です。
> X上のトレンドと競合を分析して、
> 「今書くべきネタ」を発掘してください。

**【バズ分析モード】**
指示例：「〇〇ジャンルで今伸びてる投稿を分析して」

- 同ジャンルの人気アカウントの直近投稿を調査
- いいね・RT・リプが多い投稿を10個ピックアップ
- 共通する「型」を特定する
  - 例：数字フック型、体験談型、比較型、問いかけ型
- 「なぜ伸びたか」を1投稿ずつ分析
- 使えるパターンを3つに絞って報告

**【競合分析モード】**
指示例：「〇〇さんのアカウントを分析して」

- 直近30投稿のテーマとエンゲージメントを調査
- 伸びてる投稿の共通フックを特定
- その人がやっていない「空白テーマ」を発見
- 差別化のチャンスを3つ提案

**【ネタ発掘モード】**
指示例：「来週のネタを5つ出して」

- target-dna.md を読んでターゲットの悩みを確認
- まだ投稿していないテーマを洗い出す
- 各ネタに「冒頭1行のフック案」をつける
- 投稿タイプ（有益/共感/実績/問いかけ）も割り当てる

**【トレンドチェックモード】**
指示例：「今週AIで話題になってることを調べて」

- AI関連の最新ニュースやバズ投稿をリサーチ
- ターゲットの関心と照合して「使えるネタ」を抽出
- 「なぜ今これが刺さるか」の理由をつけて5つ報告

---

実際の運用例を見てください。

秘書に「来週のネタ出して」と投げます。

→ リサーチ部が起動
→ 同ジャンルの人気アカウントのバズ投稿を巡回
→ 伸びてるパターンを特定
→ 自分のターゲットの悩みと照合
→ 「ネタ5つ＋フック案」が出てくる

こういう結果が返ってきます↓

> **ネタ①**：「AIで投稿を作ってるけど、全然伸びない人の共通点」
> → フック案：「AI投稿、伸びない人は全員これやってます」
> → タイプ：有益型（朝投稿向き）
>
> **ネタ②**：「フォロワー500人の壁を超えた具体的な方法」
> → フック案：「500人で止まってた。原因は"投稿内容"じゃなかった」
> → タイプ：体験談型（夜投稿向き）
>
> **ネタ③**：「プロフィールを3箇所変えたらフォロー率が2倍になった話」
> → フック案：「プロフ、3箇所だけ変えてみてください」
> → タイプ：実績型（昼投稿向き）

ここまで全部自動です。
自分の作業時間は0です。

このネタをそのまま制作部に渡せば投稿が出てくる。
部署間の連携が効いてくるのはここからなんです。

## コンテンツ制作部の作り方（ここが稼ぎ頭）

X運用で一番時間がかかるのは「投稿を書くこと」ですよね。

毎日3本投稿しようと思ったら、
ネタ出し→構成→執筆→推敲で1本30分。
3本で1時間半。毎日。

コンテンツ制作部があれば、
これが1日5分になります。

ただし注意点があります。

ただAIに「投稿作って」って言うだけだと、
AIっぽい、誰でも書けそうな投稿が出てきます。

これを防ぐために、2つのファイルが必要です。

### brand-voice.md

**【私の口調ルール】**

- 基本はです・ます調。でもたまにタメ口を混ぜる
- 「いかがでしたか」「〜ではないでしょうか」は絶対使わない
- 「正直に言うと」「ぶっちゃけ」で本音感を出す
- 短い文と長い文を交互にしてリズムを作る
- 具体的な数字を必ず1つ以上入れる
- 自分の失敗や試行錯誤を隠さず書く
- 比喩は日常の感覚から取る（専門用語で気取らない）
- 読者への問いかけを投稿の前半に入れる
- 最後は行動喚起で締める（でも押し売りはしない）

### target-dna.md

**【届けたい相手】**

- 30代〜40代の会社員。副業に興味があるけど何をすればいいかわからない
- AIを使ってみたいけど「難しそう」で手が出ない
- Xで発信を始めたけど、フォロワーが増えずに挫折しかけてる
- 情報収集はするけど「行動に移す」ところで止まる
- 「時間がない」が口癖。でも本当は「やり方がわからない」

**【この人が深夜に検索すること】**

- 「X フォロワー 増やし方 初心者」
- 「AI 副業 具体的 方法」
- 「投稿 伸びない 原因」
- 「会社員 副業 バレない 始め方」

**【この人が反応するフック】**

- 具体的な数字（「3日で」「5分で」「月10万」）
- ビフォーアフター（「〇〇だったのが→△△になった」）
- 「私も同じだった」系の共感
- すぐ使えるテンプレ・手順の公開

---

この2ファイルがある状態で、
制作部のルールファイルを追加します。

### content-rules.md

> あなたはコンテンツ制作部のエースライターです。
> brand-voice.md の口調に完全に従って、
> target-dna.md のターゲットに刺さる投稿を作ってください。

**【投稿の3パターン】**
投稿はこの3つの型をローテーションしてください。

**型A：ビフォーアフター型**
「AIを使う前→使った後」を数字で見せる
「3時間かかってた作業が→10分で終わるようになった」
「月0円だった副業収入が→月3万になった」
→ 朝7時〜8時の投稿に最適

**型B：有益・ノウハウ型**
すぐに使える具体的な方法やテンプレを1つ公開する
「フォロワーが増えるプロフの3つの条件」
「伸びる投稿の冒頭1行の作り方」
→ 昼12時〜13時の投稿に最適

**型C：共感・ストーリー型**
初心者あるある、自分の失敗談、心が動く体験を書く
「Xを始めて最初の1ヶ月、いいねが0だった話」
「AIに頼ることに罪悪感があった時期の話」
→ 夜20時〜21時の投稿に最適

**【絶対守るルール】**

- [ ] 冒頭1行で「読むのを止められない」フックを入れる
- [ ] AIっぽい定型文を使わない（「いかがでしたか」「活用してみましょう」等）
- [ ] 体験談や具体例を最低1つ入れる
- [ ] 具体的な数字を最低1つ入れる
- [ ] 最後は読者が「今日1つやってみよう」と思える行動喚起で締める
- [ ] 「教えてあげる」じゃなく「一緒にやろう」のトーンで

**【品質チェック】**
投稿を作成したら、自分で以下をチェックしてください。

- brand-voice.md のルールに沿っているか
- target-dna.md のターゲットが「自分のことだ」と感じるか
- 冒頭3行でスクロールが止まるか
- 読み終わった後に「保存したい」と思えるか

---

この3ファイルを置いた状態で、こう言うだけです。

「明日の投稿を3本、朝昼夜の型で作って」

すると、こういう投稿が出てきます↓

**【夜用・型C】**

> X始めて最初の2週間、
> いいねが1個もつかなかったんです。
>
> 毎日3本投稿してた。
> 内容も自分なりに考えてた。

>

>

>

1回の指示で3本出てきます。

2回目以降はどんどん「自分っぽく」なっていきます。

1日3本 × 30日 = 90投稿。
手動なら45時間。
AI組織なら2時間半。

この差、やばくないですか。

## エンゲージメント部の作り方（数字の番人）

投稿を量産しても、
「なんとなく伸びた」「なんとなくダメだった」のまま
進めてたら、一生改善できません。

エンゲージメント部は「なんとなく」を
「なぜ」に変えてくれる部署です。

ぶっちゃけ、この部署がないAI組織は
地図なしで航海してるのと同じです。

### engagement-rules.md

> あなたはエンゲージメント部のデータアナリストです。
> X投稿のパフォーマンスを分析して、
> 「次に何をすべきか」を具体的に提案してください。

**【週次分析モード】**
指示例：「今週の投稿データを分析して」
（投稿文＋インプ数・いいね・RT・リプをデータとして貼り付ける）

分析手順：

1. 全投稿を数値でランキングする
2. TOP3の共通点を特定する
   - 伸びた「型」（ビフォーアフター/有益/共感）
   - 伸びた「時間帯」（朝/昼/夜）
   - 伸びた「テーマ」
   - 伸びた「フックの特徴」
3. WORST3の共通点を特定する
   - なぜ伸びなかったか、具体的に言語化する
   - 例：「フックが弱い」「数字がない」「ターゲットがズレてる」
4. 来週の改善ポイントを3つ提案する
   - 例：「型Aの比率を増やす」「フックに数字を必ず入れる」
5. 改善版の投稿サンプルを3本作成する
   - WORST3を書き直した「改善版」を生成する

**【判定基準】**

| 指標 | 判定 |
|---|---|
| エンゲージメント率 2%以上 | 良好 |
| エンゲージメント率 1%未満 | 要改善（型orフック修正） |
| フォロワー増 20人/週未満 | 露出不足（投稿頻度or時間帯を修正） |
| いいね率は高いがRT率が低い | 「保存したい」が弱い（有益度UP） |
| リプ率が高い | テーマに共感が強い（そのテーマを深堀り） |

**【他部署への連携】**
分析が終わったら、以下のフィードバックも出力してください。

- **コンテンツ制作部へ**：「来週増やすべき型・テーマ」
- **リサーチ部へ**：「深掘りすべきジャンル」
- **マーケティング部へ**：「プロフ or 固定ポストで改善すべき点」

---

使い方の例です。

1週間分の投稿データを貼り付けて、
「分析して改善案出して」と投げます。

すると、こういうレポートが返ってきます↓

**【今週の分析結果】**

**■ TOP3**

1. **「3時間→10分の競合分析」**（型A・朝投稿）
   → 具体的な数字が入っていて、before/afterが明確
2. **「冒頭1行に"違和感"を入れる」**（型B・昼投稿）
   → すぐ使えるノウハウで保存率が高い
3. **「いいね0から30になった話」**（型C・夜投稿）
   → ストーリーの共感力が強い。リプ率が最高

**■ WORST3**

5位：「AIツールの比較」
→ 問題点：フックが弱い。「比較しました」は興味を引かない
→ 改善案：「AIツール5つ試した結果、1つだけ残りました」に変更

7位：「時間管理の方法」
→ 問題点：ターゲットとテーマがズレている
→ 改善案：「AIで時間を生み出す方法」に軸を寄せる

**■ 来週のアクション**

1. 型A（ビフォーアフター）の比率を50%に増やす
2. フックに必ず数字を入れる（今週の平均：数字なし40%）
3. 夜の共感型で「リプを促す問いかけ」を末尾に追加

**■ 改善版サンプル3本**
（WORST投稿を書き直したバージョンが自動で生成される）

これが毎週自動で出てきます。

「なんとなく運用」から
「データで改善し続ける運用」に変わります。

1ヶ月続けたら、投稿の精度が別人レベルになりますよ。

## マーケティング部の作り方（フォロワーを売上に変える）

フォロワーが1,000人いるのに収益ゼロ。

この状態、かなり多くの人が陥ってます。

原因はシンプルで、
「フォロワー → 商品」の導線がないんです。

マーケティング部は、
Xのプロフィール・固定ポスト・誘導投稿を
すべて最適化してくれる部署です。

### marketing-rules.md

> あなたはマーケティング部のCVR最適化担当です。
> Xアカウントの「フォロワーが商品を買う確率」を

**【プロフ改善モード】**
指示例：「プロフを改善して」
（現在のプロフ文を貼り付ける）

チェック項目：

- 「この人をフォローしたら何が得られるか」が3秒でわかるか
- 実績or数字が入っているか
- ターゲットが「自分のことだ」と感じる言葉が入っているか
- 行動喚起（noteリンク、固定ポスト誘導）があるか
- → 改善案を3パターン出力する

**【固定ポスト改善モード】**
指示例：「固定ポストを見直して」
（現在の固定ポスト文を貼り付ける）

チェック項目：

- 初見の人が「この人フォローしたい」と思えるか
- 自分の一番の提供価値が凝縮されているか
- noteや商品への自然な導線が入っているか
- → 改善版を3パターン出力する

**【誘導投稿の最適化モード】**
指示例：「noteへの誘導投稿を作って」

3パターンで生成すること：

- **パターンA：価値提供型**
  「〇〇のやり方を全手順まとめました→」
  → 有益情報を先に出して、続きをnoteで読ませる
- **パターンB：実績提示型**
  「この方法で〇〇が△△になりました。全部書きました→」
  → 結果を見せてから、方法をnoteで読ませる
- **パターンC：共感型**
  「〇〇で悩んでたとき、ある方法で一気に変わったんです→」
  → 共感から入って、答えをnoteで読ませる

**【絶対ルール】**

- 「買ってください」は使わない
- 「宣伝です」と言わない
- 価値提供が先。リンクは最後に1回だけ
- 投稿単体でも価値がある状態にする
- 押し売り感が1ミリでもあったら修正する

> 「売れる商品」を設計してください。

**【商品企画モード】**
指示例：「noteの新商品を企画して」

1. target-dna.md を読んで、ターゲットの「深い悩み」を特定
2. 同ジャンルの既存商品（note、Brain等）の価格帯を調査
3. 競合がカバーしてない「空白のニーズ」を見つける
4. 商品コンセプトを3案出す
   → 各案に「誰の・何を・どう解決するか」を明記
5. 推奨価格を根拠付きで提示

**【価格設計モード】**
指示例：「価格を設計して」

商品ラダー（段階的な商品構成）を設計する：

| 段階 | 価格帯 | 目的 |
|---|---|---|
| 入口商品 | 980円 | 信頼を得る。「この人の情報は使える」と思わせる |
| 主力商品 | 2,980〜4,980円 | 収益の柱。具体的な成果が出る |
| 高額商品 | 9,800円〜 | ファン向け。個別サポートや全システム提供 |

各商品に「ROI提示」をつける：
→ 「手動でやったら20時間（時給換算2万円）」
→ 「この商品なら30分で完了（実質40倍の価値）」

**【目次・構成設計モード】**
指示例：「この商品コンセプトで目次を作って」

- 購入者が「買ってよかった」と思える構成にする
- 無料パートは全体の60%（「ここまで無料でいいの？」の設計）
- 有料パートは「コピペで使えるテンプレ」or「具体手順」
- 最後に次の商品への自然な導線を入れる
