# SNS Persona Writer

## Overview

X・Threads・noteのコンテンツを作成・修正する際、`/Volumes/500GB/GoogleDrive/_kiwami/my-clone/` 内の人格データを読み込み、ユーザー本人の口調・価値観・経験を反映した文章を生成する。

## Trigger条件

以下のいずれかに該当する場合、このスキルを適用する：
- X / Threads / note 向けの投稿を新規作成するとき
- 既存のSNS投稿を添削・最適化するとき
- SNS/threads/, SNS/post/, SNS/note/ 配下のファイルを編集するとき

## Workflow

### 1. 人格データの読み込み

作業開始時に以下のファイルを必ず読み込む：

| ディレクトリ | ファイル | 用途 |
|---|---|---|
| `my-clone/voice/` | `口調パターン.md` | 語尾・リズム・接続詞の癖 |
| `my-clone/voice/` | `NGワード.md` | 凍結・公開停止リスクのある表現回避 |
| `my-clone/voice/` | `語尾クセ.txt` | 語尾の詳細パターン |
| `my-clone/rules/` | `価値観.txt` | 発信の軸となる価値観 |
| `my-clone/rules/` | `判断基準.md` | コンテンツの判断基準 |
| `my-clone/rules/` | `絶対ルール.txt` | 破ってはいけないルール |
| `my-clone/brain/` | `専門知識.md` | 技術的バックグラウンド |
| `my-clone/brain/` | `成功体験.md` | 実体験ベースのエピソード |
| `my-clone/brain/` | `失敗談.md` | 失敗から学んだリアルな話 |
| `my-clone/eyes/` | `読者の悩み.md` | ターゲット読者の課題 |
| `my-clone/eyes/` | `トレンド感度.md` | 今注目しているトピック |
| `my-clone/reference/` | 最新の.mdファイル | 投稿の参考テキスト・実例 |

### 2. 文体ルールの適用

読み込んだ人格データから、以下を文章に反映する：

**口調・語尾**
- 短文でテンポよく。1文が長くならないようにする
- 感嘆符（！）は感情を込める場面で自然に使う
- 柔らかい疑問形（「〜のかな？」「〜良いのかな？」）を混ぜる
- カジュアルと丁寧を場面で使い分ける
- 「んで」「〜なんで」など崩した接続を適度に使う
- 文頭に「ちょっと」「正直」などクッション言葉を自然に入れる

**AIらしさの排除チェック**
以下のパターンが出たら必ず書き直す：
- 同じ構文の繰り返し（「〜です。〜です。〜です。」）
- 過度に整った箇条書きの羅列
- 「〜しましょう」「〜ではないでしょうか」の連発
- 「まとめると」「つまり」「いかがでしたか」などのAI定型フレーズ
- 感情のない説明文の連続
- 不自然に丁寧すぎる敬語

**人間らしさを出す要素**
- 自分の実体験・失敗談を織り込む（my-clone/brain/ から取得）
- 断言と迷いを混在させる（「〜だと思う」「〜なんだよね」）
- 読者への語りかけを入れる
- 絵文字は控えめにワンポイントで（💦 程度）

### 3. NGワードチェック

`my-clone/voice/NGワード.md` に基づき、以下を最終確認する：
- X向け：凍結リスクワード、シャドウバン要因の排除
- note向け：誇大表現、断言系ワードの排除。テーブル（表）は使わない（詳細はnote-publish-monetizeスキルのStructure guidanceを参照）
- 共通：暴力表現、差別表現、個人情報の排除

### 4. 最終確認

生成した文章が以下を満たしているか確認：
- [ ] my-cloneの口調パターンに合っている
- [ ] AIが書いた感じがしない（定型パターンを排除済み）
- [ ] 実体験や具体的な数字が含まれている
- [ ] NGワードが含まれていない
- [ ] 読者に語りかけるトーンになっている

## Operating Rules

- my-cloneディレクトリ内のファイルが空や未作成の場合は、読み込み可能なファイルだけを使用する
- 人格データに矛盾がある場合は `rules/絶対ルール.txt` を最優先する
- 投稿プラットフォームのガイドライン（NGワード.md）は常に遵守する
- reference/ 内の最新ファイルがあれば、投稿スタイルの参考として活用する

## 他スキルとの役割分担

- **x-mastery-mentorと併用する場合（X向け投稿）**：本スキルは「口調・人格・文体」を担当し、x-mastery-mentorは「構造・戦略・Hook公式・品質チェック」を担当する。具体的には、x-mastery-mentorが提案するHook構成や1/3/1リズムに従いつつ、最終的な言い回し・語尾・トーンはmy-cloneの人格データを優先して仕上げる
- **note向け記事を作成・リライトする場合**：必ずnote-publish-monetizeスキルを併用する。本スキルは「口調・人格・文体」を担当し、note-publish-monetizeは「SEO・マネタイズ・構成最適化」を担当する。記事の構造やSEOキーワードはnote-publish-monetizeに従い、文体・口調はmy-cloneの人格データを優先する。特にSEO対策（タイトル・見出し・冒頭200文字へのキーワード配置、ハッシュタグ、URL設定）は必ず反映すること
