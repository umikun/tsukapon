## 📝 候補エントリのフォーマット

新しい候補を追加するときは以下の形式で書く：

```markdown
### {スキル候補名（仮）}

- **検出日時**: YYYY-MM-DD HH:MM（最新の検出。1回目の日付も保持）
- **検出回数**: N
- **トリガー条件**: ユーザーが「〜」と言ったとき / 〜のパターンが現れたとき
- **典型例**:
  - YYYY-MM-DD: {そのときの会話・タスクの要約}
  - YYYY-MM-DD: {〃}
- **実装案（ドラフト）**:
  - 入力: 〜
  - 処理: 〜
  - 出力: 〜
- **既存スキルとの関係**: {重複するか / 連携するか / 完全新規か}
- **メモ**: {実装するか迷う理由・判断軸}
```

## 🧭 候補追加の判断軸

以下のいずれかを満たすパターンが現れたら候補に追加：

1. **頻度**: 同じ作業（命名規則・配置先・処理フロー）が直近1ヶ月で3回以上
2. **明示的依頼**: ユーザーが「これスキルにしたい」と言った
3. **作業量**: 毎回30秒以上かかる手作業がある
4. **再現性**: 入力 → 出力のマッピングが安定して書ける

逆に**候補にしないケース**:

- 1回限りの特殊作業
- ユーザーの判断が毎回変わる（自動化できない）
- 既存スキルの軽い拡張で済む（→ 既存スキルを更新）
