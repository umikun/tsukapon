## 3. A2後も残る痛みポイント（B案/C案で対応する候補）

### 3-1. SNS運用ルートに散らばる13本のドキュメント

```
Discordについて / Grok運用ネタ出し術 / OGP画像サイズ一覧 /
Obsidian Web Clipper活用 / Threads運用 / growth-strategy /
memo / name / noteの今後と収益化戦略 / note記事用サムネイル仕様書 /
サムネイル画像 生成手順 / 仕事効率爆上げジャーナリングテンプレ /
日常的に使っているツール
```
→ runbook / spec / scratch / template が混在し、見分けがつかない

### 3-2. サムネイル関連の分裂

- `サムネイル作成/` — 素材（.ai, .jpg）
- `title/` — 文言テキスト
- `note記事用サムネイルのデザインシステム仕様書.md` — ルートに散在

### 3-3. 同じ日の関連ファイルが**6箇所に分散**

4/27 を例にすると：

| 種類 | 現在地 |
|---|---|
| X日次 | `SNS運用/post/day95.md` |
| Threads日次 | `SNS運用/threads/Threads-day95.md` |
| note記事本体 | `SNS運用/note/note-20260427.md` |
| X Articles版 | `SNS運用/post/article/X-20260427.md` |
| note紹介スレッド | `SNS運用/post/thread/20260427_note紹介スレッド.md` |
| サムネ文言 | `SNS運用/title/title-20260427.md` |

### 3-4. 命名規則の3系統混在

- 連番: `day61.md` 〜 `day95.md`（post/）
- 連番＋プレフィックス: `Threads-day61.md`（threads/）
- 日付: `note-20260427.md` / `X-20260427.md` / `title-20260427.md`

→ `day95` と `note-20260427` が同じ日のものか即座に判別不可
