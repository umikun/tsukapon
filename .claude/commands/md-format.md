選択したテキスト、またはファイル全体のマークダウン記法を検査・修正するスキルです。

## 入力形式

以下の2パターンに対応します：

**パターン①：ファイル全体を整形**
```
ファイルパス
```

**パターン②：選択テキストを整形**（`<editor_selection>` タグがある場合）
- ファイルパスと選択範囲（行番号）を自動取得
- 選択範囲のみを修正し、ファイルの該当箇所に上書き

入力なし（`$ARGUMENTS` が空）かつ `<editor_selection>` がある場合は、自動的にパターン②として動作する。

## 処理手順

### Step 1: 対象テキストの取得

- パターン①の場合: 指定ファイルを全文読み込む
- パターン②の場合: `<editor_selection>` の内容を対象とする。ファイルパスと行番号を記録しておく

### Step 2: マークダウン問題の検査

以下の観点で問題箇所を列挙する：

#### コードブロック
- 開始 ` ``` ` に対応する閉じ ` ``` ` があるか
- ネストが正しいか（コードブロック内にコードブロックが入っていないか）
- 言語指定（` ```python ` 等）の有無（任意。あった方が読みやすい場合は補完を提案）

#### 見出し
- h1（`#`）が複数あってドキュメント構造が壊れていないか
- 見出しレベルが飛んでいないか（h2の次にいきなりh4など）
- 見出しの前後に空行が1行あるか

#### リスト
- 箇条書きの記号が混在していないか（`-` と `*` が混在）
- インデントの深さが一貫しているか（半角スペース2または4で統一）
- 番号付きリストの番号が正しい順序か

#### 強調・装飾
- `**`（太字）の開閉が対応しているか
- `*`（イタリック）の開閉が対応しているか
- バックティック（`` ` ``）のインラインコードが正しく閉じているか

#### テーブル
- ヘッダー行 → 区切り行（`|---|`）→ データ行の3段構成になっているか
- 列数が全行で一致しているか
- 区切り行に `-` が最低1個あるか

#### その他
- 過剰な空行（3行以上連続）を2行以内に詰める
- 行末の不要なスペース（トレーリングスペース）
- リンク記法 `[テキスト](URL)` の括弧閉じ忘れ
- 引用ブロック（`>`）の前に空行があるか

### Step 3: 修正の実施

- 検出した問題を全て修正した「整形後テキスト」を生成する
- **元の内容・意味は一切変えない**。あくまで記法の修正のみ
- 修正箇所を変更前/後のdiff形式で、修正理由とともにチャット内に明示する

### Step 4: ファイルへの保存

**パターン①（ファイル全体）の場合**:
- 修正後テキストをファイルに上書き保存する

**パターン②（選択範囲）の場合**:
- ファイルの該当行（行番号を参照）を修正後テキストで差し替えて保存する

### Step 5: 完了報告

修正した問題の件数と種類を以下の形式で報告する：

```
✅ マークダウン整形完了
- コードブロック閉じ忘れ: 1件修正
- 過剰な空行: 2件修正
- （問題がなければ「問題は見つかりませんでした」と報告）
```

## 制約事項

- 文章の内容・意図・語順は**絶対に変えない**。記法のみ修正する
- Obsidianの独自記法（`![[画像]]`、`[[ページ]]`、`%%コメント%%`）は正常記法として扱い、変更しない
- Dataviewクエリブロック（` ```dataview ` ）は内容を変更しない
- 修正が0件の場合でも「問題は見つかりませんでした」と報告する

## 入力データ

$ARGUMENTS
