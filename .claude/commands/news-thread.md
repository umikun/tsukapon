AIニュースを「今週の1本」に絞り、X/Threads向けの**v2型スレッド（5投稿構成）**を生成するスキルです。
火・金の週2回モデル（[[SNS運用/analytics/news_thread_v2.md]]）を実行装置として動かす役割。

## 🚨 必須パイプライン宣言

このスキルは Step 1〜9 をノンストップで完走する。Step 6 で応答を切らず、Step 9 の保存・検証まで一気に走り切ること。

## 入力形式（4パターン対応）

### A. ニュース直書き（最頻パターン）

```
/news-thread
【今週の1本】OpenAI が o4-mini を発表
【ソース】https://openai.com/blog/...
【自分の現場】社内でo3 → o4-mini に切替検証始まってる
（任意）【曜日】fri  ← tue / fri。省略時は本日の曜日から自動判定
（任意）【テーマslug】o4-mini  ← ファイル名用。省略時は自動生成
```

### B. 調査ノート起点

```
/news-thread
【ソース】調査/2026-04-26-claude-code-100-best-repos.md
【自分の現場】煽りに引きずられた人を週3で見てる
```

→ ソースから1本のニュースとして主題を抽出

### C. note記事/アーカイブ起点

```
/news-thread
【ソース】SNS運用/note/note-20260427.md  ← または SNS運用/archive/note/ 配下
```

→ note記事の中から最も尖ってる1本を選び、批評型に再構成

### D. 引数なし（対話モード）

```
/news-thread
```

→ 「今週のソース候補出しますね」と聞き、`調査/` `SNS運用/note/` `SNS運用/archive/note/` から直近1週間のファイルを列挙して選ばせる

4. `SNS運用/analytics/WNN戦略メモ.md`（当週方針 / 例: [[SNS運用/analytics/W18戦略メモ.md]]）

## 処理手順（9ステップ）

### Step 1. 入力パターンの判定とソース確定

- A/B/C/D のいずれかを判定
- 「今週の1本」のニュース主題を1文に確定（20字以内の見出し候補も用意）
- ソースが調査ノート or note記事の場合、本文を読んで主題を抽出
- D（対話）の場合、`調査/` `SNS運用/note/` `SNS運用/archive/note/` から `ls -t` で直近1週間のファイル名を列挙して提示

### Step 2. 曜日判定と「角度」決定

- 火曜（tue）: 「先週末の動きを束ねて1本」→ 主題は「週明けにこれを知ってると差がつく」
- 金曜（fri）: 「週中の最大トピックを1本」→ 主題は「週末までにこれを試しておくと月曜が変わる」
- 入力で曜日指定がなければ `date +%u` で判定。火金以外なら**警告を出した上で**続行（「v2の本来運用は火金です」と1行報告）

### Step 3. 戦略コンテキストの照合

- 当週 `WNN戦略メモ.md` があれば「3つのKPI」「7日間カレンダー」を読み、本日のスレッドに期待される役割を確認
- 前週 `WNN分析レポート.md` があれば、伸びた型・刺さったフックを抽出して反映候補にする

### Step 4. 1/5（フック・要約）の生成

ルール（テンプレ準拠・**全項目必須**）：

- [ ] 1文目に希少性訴求語: 「効くの1本」「実は」「本当は」のいずれか
- [ ] 主題見出しは **20字以内**
- [ ] 自分の現場 or 同僚事例を1つ匂わせる（`【自分の現場】`入力 or 推定）
- [ ] 絵文字 **1個** （多すぎ禁止）
- [ ] 語尾は断定しない（「〜気がする」「〜と思ってる」「〜らしい」）
- [ ] 末尾に「順に解説します👇」 or 同等の続き予告
- [ ] 140字以内

### Step 5. 2/5（一般論じゃない見解）の生成

- [ ] 冒頭に「ほとんどの解説が〇〇で止まってる」フレーズで批評ポジを匂わせる
- [ ] 「実際は **〇〇な業務をしてる人ほど不利になる** 構造があると思ってて」型の構造提示
- [ ] 理由を**正確に3つ**（4つ以上禁止）。①②③の番号付き
- [ ] 140字以内
- [ ] my-clone口調（です・ます基調＋カジュアル崩し）

### Step 6. 3/5（読者の不安を1つ植える）の生成

- [ ] **関係性キーワード必須**: 「上司」「同僚」「会社」「取引先」「部下」のいずれか最低1つ
- [ ] 「で、これ放置すると来月以降、こうなります：」型の導入
- [ ] ❌マークで不安を**正確に3つ**
- [ ] 「特に最後のやつ、地味に効いてくる気がする😭」型の落とし
- [ ] 絵文字 **1個**
- [ ] 140字以内

⚠️ **このステップで応答を切らない**。Step 7〜9 まで続ける。

### Step 7. 4/5（具体的アクション）の生成

- [ ] 「じゃあ何すればいいか、今週中の最小アクション：」型の導入
- [ ] ✅マークでアクションを**正確に3つ**
- [ ] **全アクションが15分以内に収まる**こと
- [ ] 動詞は「読む」「ブクマ」「試す」のいずれかで構成（実行コスト低めに統一）
- [ ] 末尾に「全部15分以内。週末に詰め込めば終わります」型のクロージング
- [ ] 140字以内

### Step 8. 5/5（誘導＋note）の生成

- [ ] 「詳細はnoteにまとめました」型の導入
- [ ] **当日のnote記事URL** を `🔗` 付きで記載
  - 入力で `【ソース】SNS運用/note/note-YYYYMMDD.md` を受けた場合、本文先頭の `[note記事URL]` 行を抽出
  - 抽出できない / note起点でない場合は仮URL `https://note.com/chackwill/n/abc` を入れて「公開後に手動差し替え」と備考
- [ ] プロフ誘導文を必ず1ブロック（「このアカウント、AI界隈の煽りに毎朝ツッコみつつ自分の仕事に効く部分だけ抜き出す人なので、フォローしてもらえると次のスレッドも届きます」型）
- [ ] **ハッシュタグ禁止**（v2はbot認識回避のため削除）
- [ ] 140字以内（URLは字数外）

### Step 9. 検証・保存・報告

#### 9-1. 自動検証（5項目）

以下が **全て満たされている** ことを `Read` で確認してから報告：

- [ ] 5投稿すべて 140字以内
- [ ] 1/5 と 5/5 が `---` 区切りで明確に分離
- [ ] 3/5 に関係性キーワードが1つ以上含まれる
- [ ] 4/5 のアクションが3つ・全部15分以内
- [ ] ハッシュタグが**含まれていない**

#### 9-2. 保存

- 保存先: `SNS運用/post/thread/YYYYMMDD_news_v2_{slug}.md`
  - `YYYYMMDD` は本日の日付
  - `{slug}` は入力 or 主題から自動生成（英数とハイフン）
- フォーマット: 各投稿を `---` で区切り、`### 1/5（フック・要約）` 等の見出しを必ず付与
- 冒頭に H1 タイトルと **`🔗 関連コンテンツ` callout** を付与（CLAUDE.md 自動リンクルール準拠）
  - 必須リンク: テンプレ・当週戦略メモ・ソースファイル

#### 9-3. 完了報告

チャット内に以下を含める：

1. 保存先 wikilink
2. **コピペ用の5投稿本文**（**1投稿につき1コードブロック**で5本独立して出力。`---` 区切りはコードブロックの外）
3. 各投稿の字数
4. 採用した型・関係性キーワード
5. note URL を実URLに差し替える必要があるかどうかの注意書き
6. （CLAUDE.md準拠）「📋 スキル一覧の更新は不要」or「不要」を1行
7. （CLAUDE.md準拠）`.claude/commands/` 配下のため自動リンクルール対象外であることは宣言不要

## 制約・禁止事項

- **ハッシュタグ禁止**（v2はbot認識回避のため）
- **絵文字は1投稿に最大2個**（テンプレ準拠）
- **「いかがでしたか」「活用してみましょう」等のAI定型句禁止**（CLAUDE.md準拠）
- **web制作者であることは記述しない**（CLAUDE.md準拠）
- 自動投稿は提案しない（手動投稿前提）
- 既存の `SNS運用/post/thread/YYYYMMDD_news_v2_*.md` がある場合は上書きせず、ファイル名末尾に `_2` を付ける

## 既存スキルとの関係

| スキル | 違い |
|---|---|
| [[.claude/commands/thread.md|/thread]] | note記事 → 8投稿スレッド変換用。1記事を網羅する設計 |
| **`/news-thread`（本スキル）** | 1ニュース → 5投稿スレッド。深さ3倍で1本に絞る設計 |
| [[.claude/commands/re-daily.md|/re-daily]] | note記事リライト＋4ファイル生成。News Thread とは独立 |

`/re-daily` の Step 9（Xスレッド生成）は note記事ベースなので、`/news-thread` と二重実行はしない。火金朝は `/news-thread` 単体で動かす想定。
