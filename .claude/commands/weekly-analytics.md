X運用の週次データ分析スキル。投稿のパフォーマンスを数値で評価し、次週の改善アクションと改善版サンプルを生成する。

## 入力形式

以下のいずれかの形式で受け取る：

### パターンA: データ直接貼り付け

```
週次データ：YYYY-MM-DD 〜 YYYY-MM-DD
---
投稿1:
[投稿本文]
インプ: 1234 / いいね: 45 / RT: 3 / リプ: 2 / 保存: 5 / プロフアクセス: 20
---
投稿2:
...
```

### パターンB: ファイルパス指定

```
データファイルパス
例: SNS運用/analytics/W16分析レポート.md
```

**Xアナリティクス CSVを使う場合**: 生CSVは `SNS運用/analytics/source/` に保存する。
- `account_analytics_content_YYYY-MM-DD_YYYY-MM-DD.csv` … 投稿別データ
- `account_overview_analytics_YYYY-MM-DD_YYYY-MM-DD.csv` … 週次サマリー

レポート本文（`2026-WNN.md`）からは `[[SNS運用/analytics/source/...csv]]` でwikilink参照する。

### パターンC: 空入力（対話モード）

入力が空の場合、以下を順に質問する：
1. 分析対象期間（例: 今週 / 2026-04-11〜04-17）
2. 投稿データ（本文＋指標をまとめて貼ってもらう）
3. 現在の総フォロワー数・前週からの増減

## 処理手順

### Step 1: 情報の読み込み

1. 入力データを読み込む
2. `SNS運用/post/フォロワー改善.md` を読んで、現在の課題と改善方針を把握する
3. `_ kiwami/my-clone/` 内の人格データを参照し、改善版投稿の文体を揃える
4. `SNS運用/post/day*.md` の最新10本程度にざっと目を通し、直近のトーン・構成を把握する

### Step 2: 数値分析

各投稿について以下を計算する（データが揃っている範囲で）：

- **エンゲージメント率** = (いいね + RT + リプ + 保存) ÷ インプ × 100
- **プロフィールアクセス率** = プロフアクセス ÷ インプ × 100
- **保存率** = 保存 ÷ インプ × 100（「有益度」の指標）
- **RT率** = RT ÷ インプ × 100（「拡散力」の指標）
- **リプ率** = リプ ÷ インプ × 100（「共感・議論誘発度」の指標）

### Step 3: TOP3 / WORST3 の抽出

**TOP3**: エンゲージメント率で上位3つ。各投稿について以下を分析：

- 伸びた「型」: ビフォーアフター / 有益ノウハウ / 共感ストーリー / 問いかけ / 実績提示
- 伸びた「時間帯」: 朝（7-9時）/ 昼（12-14時）/ 夜（20-22時）/ その他
- 伸びた「テーマ」: AIニュース / 副業 / 時短術 / プロンプト術 / 実体験 など
- 伸びた「フックの特徴」: 具体的な数字 / 違和感 / 逆張り / 権威 / 失敗談 など
- なぜ伸びたか、1文で言語化

**WORST3**: エンゲージメント率で下位3つ。各投稿について：

- なぜ伸びなかったか、具体的に言語化（「フックが弱い」「数字がない」「ターゲットがズレている」等）
- 改善の方向性を1文で提示

### Step 4: 総合判定

以下の判定基準で総合評価を出す：

| 指標 | 良好 | 要改善 |
|---|---|---|
| エンゲージメント率（週平均） | 2.0%以上 | 1.0%未満 |
| プロフィールアクセス率 | 0.5%以上 | 0.2%未満 |
| フォロワー増（週次） | 20人以上 | 5人未満 |
| 保存率 | 0.3%以上 | 0.1%未満 |

**判定パターン**:

- いいね率◎ RT率× → 「感情は動くが拡散されない」（=保存したくなる有益度を上げる）
- いいね率× RT率◎ → 稀。ほぼ起きない
- インプ◎ プロフアクセス率× → **フォロー導線が壊れている**（最優先課題）
- リプ率◎ → テーマに共感が強い。**そのテーマを来週深掘り**
- 全般的に低迷 → 型かターゲットがズレている可能性。ターゲットDNAを見直す

### Step 5: 来週のアクションプラン（3つ）

Step 4の判定を元に、**実行可能な具体アクション**を3つ提示する。

- 良い例: 「型A（ビフォーアフター）の比率を50%に増やす」「フックに数字を必ず入れる」「CTAを『フォローしてください』に統一」
- 悪い例: 「もっとバズる投稿を書く」「エンゲージメントを上げる」（抽象的で実行不能）

さらに、`フォロワー改善.md` との整合性も確認：

- フォローCTAは全投稿に入っているか？
- ハッシュタグは統一されているか？
- エンゲージメント質問は末尾に入っているか？

### Step 6: 改善版サンプル3本の生成

WORST3の投稿を、以下のルールで書き直す：

- `my-clone/` の文体を反映
- 冒頭1行に「違和感」or「具体的な数字」を入れる
- 体験談 or 具体例を最低1つ含める
- 末尾にフォローCTA or エンゲージメント質問を入れる
- 140文字以内（スレッド化する場合は明記）

元投稿 → 改善版の **Before/After形式** で並べて提示する。

### Step 7: 他部署（他スキル）への連携提案

分析結果を既存スキルに流し込む提案を出す：

- **`re-daily` へ**: 「来週のnote記事は〇〇テーマが良さそう（リプ率が高かった）」
- **`thread` へ**: 「TOP1の投稿をスレッド化すると伸びそう」
- **`reply` へ**: 特になし（都度実行）
- **プロフィール見直し**: プロフアクセス率が低い場合、プロフ改善を提案

### Step 8: レポート保存

以下のパスにMarkdown形式で保存する：

```
SNS運用/analytics/WNN分析レポート.md
```

（WNN = ISO週番号。例: `W16分析レポート.md`）

**命名規則の使い分け（2026-04-27〜）**:
- `WNN分析レポート.md` … 本スキルの出力（実績の事後分析）
- `WNN戦略メモ.md` … 翌週の方針メモ（手動作成 / 別ドキュメント）
- 同週でこの2本がペアになる想定（例: `W16分析レポート.md` を踏まえて `W17戦略メモ.md` を作る）

## 制約事項

- **web制作者であることは記述しない**
- AIっぽい硬い文章にしない。カジュアルで読みやすいトーンにする
- 人格データ（my-clone）の文体を反映させる
- 数字がない抽象的なアドバイスは出さない（「頑張る」「改善する」禁止）
- 改善アクションは最大3つに絞る（多すぎると実行されない）
- データが不足している指標は「データなし」と明記し、推測で埋めない
