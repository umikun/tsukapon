直近1週間の activity データから、曜日別・時間帯別の集中傾向と脱線パターンを抽出して週次レポートを生成するスキル。**3回以上繰り返す脱線パターンを検出した場合、自己改善ループに従って [[Claudian-スキル候補.md]] にも記載**する。

## 入力形式

```
/focus-report [週指定]
```

- 引数なし or `current` → 今週（ISO週、月曜起点）
- `last` → 先週
- `YYYY-Wnn`（例: `2026-W18`） → 指定週

## 処理手順

### Step 1: 集計データの取得

```bash
/usr/bin/python3 "_ kiwami/tools/daily-log/analytics.py" weekly <週指定> > /tmp/focus-week.json
/usr/bin/python3 "_ kiwami/tools/daily-log/analytics.py" distraction-patterns --days 7 --min-occurrences 3 > /tmp/focus-distraction.json
```

両方の JSON を読み込む。

### Step 2: 集中ピーク時間帯のランキング

`focus_peaks` 上位10件から、**時間帯（hour）別に集計し直す**：

- 各ピークの開始時刻の hour（例: 09:30 → 9）でグループ化
- hour ごとに「ピーク回数 / 平均長さ」を出す
- 「最も集中ピークが出やすい時間帯トップ3」を抽出

これは `SNS運用/action-YYYYMMDD.md` の「タイムライン早見表」で **最重要枠を集中ピーク時間帯に配置する根拠**として使う。

### Step 3: 曜日別 focus 比率

`by_day` から各曜日の focus_ratio_pct を計算：

```python
focus_ratio = focus / (focus + other + distraction)  # idle除外
```

7日揃わない場合は揃った曜日だけ表示。

### Step 4: 時間帯別カテゴリ分布（ヒートマップ）

`weekday_hourly_heatmap` を読み込み、24×7 のテーブルを作る：

- 行: 0-23時
- 列: Mon-Sun
- セル: その時間帯の focus 比率（%）
- 「データなし」セルは `-` で表示

簡易版として **focus比率を絵文字の濃淡で表現**:
- 80%以上: 🟩
- 50-79%: 🟨
- 20-49%: 🟧
- 1-19%: 🟥
- 0% or データなし: ⬜

### Step 5: 脱線パターン検出 → 自己改善ループ連携

`distraction-patterns` の結果を確認：

- **3回以上検出された (hour, app) パターン** が 1つ以上ある場合:
  1. レポート本文の「⚠️ 繰り返す脱線パターン」セクションに列挙
  2. **[[Claudian-スキル候補.md]] に新エントリを追加**（既存パターンと重複しないよう確認）：
     ```markdown
     ## 🟡 候補（観察N回）
     ### {hour}時の{app}脱線（observed N days）
     - 観察日: YYYY-MM-DD, YYYY-MM-DD, ...
     - 提案: {対策案を1〜2行}
     - 出典: `/focus-report` Wnn
     ```
  3. 既に同パターンが **⚪見送り判定済み** の場合は再追加しない（CLAUDE.md「🔁 自己改善ループ」ルール）

### Step 6: レポート生成

`SNS運用/analytics/focus/Wnn-focus-report.md` に以下フォーマットで出力：

```markdown

> - 📊 Daily Log: <http://127.0.0.1:8765>

> - 📋 戦略メモ: [[SNS運用/analytics/WNN戦略メモ.md]]
> - 🧠 候補: [[Claudian-スキル候補.md]]（脱線パターン検出時）

## サマリー
```

### Step 7: 完了報告

- 出力パス（wikilink）
- 集中度スコア（先週比あれば併記）
- 集中ピーク時間帯トップ1
- 検出した脱線パターン数（あれば）
- スキル候補に追加した場合は明記

## 完了時の宣言

タスク完了報告に以下を必ず含める：
- 「📊 Focus Report を生成しました: [[SNS運用/analytics/focus/Wnn-focus-report.md]]」
- 「🎯 集中ピーク時間帯: HH時台（N回, 平均NN分）」
- 脱線パターンを検出した場合: 「🟡 [[Claudian-スキル候補.md]] にN件追加しました」
- 検出ゼロ: 「✨ 繰り返す脱線パターンなし」

## 制約事項

- 7日揃わない場合（週途中など）でも動かす。揃わない曜日は `-` で表示
- 候補ファイルへの追加は **重複防止**: 既存エントリと (hour, app) が一致するなら追記せずスキップ
- ⚪見送り判定済み のエントリは再追加しない（CLAUDE.md 自己改善ループルール）
- 数字を盛らない／推測で埋めない
