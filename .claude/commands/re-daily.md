note記事をリライト（品質改善・SEO最適化・内容充実化）し、続けてX記事転載版とXスレッド版まで一気に生成するスキルです。

---

## ⚠️ 必須パイプライン宣言（最重要）

このスキルは **Step 1 → Step 11 までを1コマンドで全て完了するパイプライン** です。以下のルールを厳守してください。

1. **Step 6（タグ選定 + 保存）の完了時点で絶対に最終応答を終えないこと。** Step 6 は中間マイルストーンであり、まだ Step 7・8・9・10・11 が残っています。
2. **Step 7（サムネイル文言生成）／Step 8（X Articles版の生成）／Step 9（Xスレッド版の生成）／Step 11（本日アクションプラン生成）は必須です。** 「オプション」「任意」ではありません。これら全ファイルが生成されるまでタスクは完了しません。
3. **最終出力は必ず5ファイル**: ①note本体、②サムネイル文言ファイル、③Article版、④Thread版、⑤本日アクションプラン。どれか1つでも欠けていれば未完了です。
4. **Step 10（完了検証と報告）で5ファイルの存在を確認してから**、最終応答を返してください。
5. 途中でユーザーに質問する／進捗確認を挟む必要はありません。**全Step完了までノンストップで走り切ってください。**

## 入力形式

記事ファイルのパスを1つ受け取ります：

```
記事ファイルパス
```

任意で、すでに公開済みの場合はnote記事URLを2行目に付与できます（Step 9のスレッド生成で使用）：

```
記事ファイルパス
note記事URL
```

※ タイトルはファイル内のh1見出しから自動取得します。
※ 有料記事リンクは固定のため入力不要です。
※ note記事URLが未指定の場合、Step 9のスレッドでは仮URL `https://note.com/chackwill/n/abc` を使用します（公開後に手動で差し替え）。

## 処理手順

### Step 1: 情報の読み込み

1. 指定された記事ファイルを読み込む
2. `SNS運用/note/` 内の過去記事を確認し、フォーマット・文体の基準を把握する
3. `_ kiwami/my-clone/` 内の人格データを参照し、文体・トーンを反映させる

### Step 2: 記事の品質チェック

以下の観点で問題を調査する：

- **重複チェック**: 過去記事と内容が重複している章・セクションがないか確認。重複があれば該当セクションを削除し、代わりとなる新しいセクションを生成する
- **事実確認**: 情報の正確性に疑義がないかチェック
- **文章品質**: 冗長な表現、論理の飛躍、読みにくい箇所の改善
- **web制作者であることは記述しない**

### Step 3: SEO最適化 + 内容充実化 + W16学習の反映

参照: [[SNS運用/analytics/W17戦略メモ.md]] と [[SNS運用/Threads運用.md]] の7つの型

1. **タイトルのSEO最適化 + 強フック化**
   - 検索されやすいキーワードを含める
   - noteのタイトル文字数制限（全角40文字程度）を意識する
   - **必須**: 以下7つの型のいずれかに当てはめる
     - 強い冒頭（命令形＋否定）/ 具体的な数字 / 常識を覆す / 対比 / 読者の痛み / 今すぐ行動 / 損失回避
   - **RT誘発**: 可能なら「上司」「同僚」「会社」などの関係性キーワードを検討
   - 数字・具体性・感情を入れてクリック率を高める

2. **本文のSEO最適化 + 1秒フック**
   - 見出し（h2/h3）にキーワードを自然に含める
   - **導入文の冒頭1文**は強い動詞で始める（「今日変わった」「〇〇が死んだ」「あなたの〇〇が」など）
   - 「昨日の続き」「〇日目」などの前提依存を削除（第一話化）
   - 内部リンクや関連情報の追加で記事を充実させる

3. **内容の充実化 + 保存誘発**
   - 追記できる情報があれば追加して記事の価値を高める
   - 各ニュースの実務観点での感想・分析を強化
   - 読者が「次に何をすべきか」が明確になるアクションを提示
   - **必須**: 各章に1つ以上のチェックリスト or 番号付きリストを入れる
   - **必須**: プロンプト例は `コピペ用` コードブロックで明記
   - **推奨**: 各章の末尾に「この章を保存して、〇〇する時に使って」の保存誘導を1文入れる

4. **参考元記事のURL明記**
   - リライト時に参考にした元記事・情報源のURLを各セクションに明記する

### Step 4: フォーマット整備

過去記事（`SNS運用/note/` 内）のフォーマットに合わせる：

- **冒頭の導入文**（タイトル直下のSEO用リード文と「みなさん、おはようございます！」から始まる挨拶導入文は内容が重複しやすいため、**1つの導入ブロックに統合する**。構成：「みなさん、おはようございます！今日のAIニュースまとめです。」で始め、昨日との接続→今日のニュース概要→テーマの提示→「今日もN本でいきましょう！」で締める。タイトル直下に別途SEO概要文を置かない）
- **目次セクション**（「この記事で分かること」と「ニュース一覧」は統合して1つの目次にする。各ニュースの見出しに「何が分かるか」の補足を添え、SEOキーワードを自然に含める。「この記事で分かること」と「今日のAIニュースN本(目次)」を別々に置かない）
- 各ニュースの構成: 概要 → 実務観点での感想 → ポイントまとめ → 情報源リンク
- セクション間の `---` 区切り
- トレンドまとめ・3レイヤー分析（該当する場合）

### Step 5: 有料記事リンクの設置

記事の最後に以下の有料記事へのリンクセクションを設置する：

```
---

## 📢 「ニュースを読んで終わり」にしたくない人へ

このnoteでは毎日AIニュースを届けていますが、**「じゃあ自分の仕事にどう活かすの？」** が本当に大事なところ。

ぶっちゃけ、ニュースを読むだけだと「すごいなー」で終わっちゃうんですよね。僕も昔そうでした。

AIを実務で使いこなすための具体テクニックは、有料記事で深掘りしています。

👇 特にこの記事は反響が大きかったので、まだ読んでない方はぜひ。

🔗 [【PDF付】月80時間の残業→ゼロ。30個のプロンプトで人生が変わった話](https://note.com/chackwill/n/nd015195d2717)

正直に言うと、**この30個のプロンプトを使い始めてから、僕の働き方は完全に変わりました**。残業が当たり前だった日々が、定時退社が当たり前になった。空いた時間で新しいことにも挑戦できるようになった。

「AIを使えるようになりたいけど、何から始めればいいか分からない」って人は、まずはこの30個のプロンプトをコピペで試してみてください。1個でも自分の仕事にハマったら、それだけで世界が変わります。

**スキ・フォローで応援してもらえると、明日も書く力になります！**
```

### Step 6: タグ選定 + 保存

1. 記事の内容に最適なタグを **5つ** 選定する
   - noteで検索されやすいタグを優先
   - 例: `#AIニュース` `#生成AI` `#ChatGPT` `#AI活用` `#業務効率化`
2. 記事末尾に `## 📌 タグ` セクションとしてタグを追記する
3. 元のファイルに **上書き保存** する

> ⏭️ **重要リマインド**: Step 6 完了時点では **絶対に最終応答を返さないこと**。
> ここはまだ1/4の進捗です。続けて Step 7（サムネイル文言生成）→ Step 8（X Articles版）→ Step 9（Xスレッド版）→ Step 10（検証と完了報告）に進んでください。

### Step 7: サムネイル文言の生成（メイン＋各見出し）

note記事のメインサムネイルと各項目見出しサムネイルに使う「メインタイトル」「サブタイトル」を選定し、別ファイルとして保存する。

#### 生成するもの

1. **メインサムネイル**（記事トップに1枚）
   - **メインタイトル**: 12〜20字。記事全体のフックを凝縮した1行
   - **サブタイトル**: 20〜35字。メインを補完する具体的なベネフィット・数値・所要時間

2. **各見出しサムネイル**（各ニュース／章ごとに1組）
   - 記事内の `## 数字. 見出し` にマッチする章すべてが対象（通常は5章）
   - それぞれに **メインタイトル（12〜20字）** と **サブタイトル（20〜35字）** を生成

#### 選定ルール（7つの型＋関係性キーワード）

参照: [[SNS運用/Threads運用.md]] の7つの型

- メインタイトルは以下の7つの型のいずれかに当てはめる：
  - ① 強い冒頭（命令形＋否定） / ② 具体的な数字 / ③ 常識を覆す / ④ 対比 / ⑤ 読者の痛み / ⑥ 今すぐ行動 / ⑦ 損失回避
- **RT誘発**: 可能なら「上司」「同僚」「会社」などの関係性キーワードを仕込む
- サブタイトルは「何が手に入るか」「何がわかるか」を明示し、メインと意味が重複しないこと
- 章の具体内容（数字・固有名詞・読者の痛み）から引き出す。**コピペや抽象的表現はNG**

#### 保存先

- `SNS運用/title/title-YYYYMMDD.md`（日付は記事の日付）
- 事前にディレクトリ不存在なら `mkdir -p "SNS運用/title"` で作成してから保存

#### ファイルフォーマット

```
# YYYYMMDD サムネイル文言

> 記事: [[SNS運用/note/note-YYYYMMDD.md]]
> 記事タイトル: <note本体のh1>

---

## 🎨 メインサムネイル（記事トップ）

- **メインタイトル**: <12〜20字>
- **サブタイトル**: <20〜35字>
- **採用した型**: <①〜⑦のいずれか>
- **関係性キーワード**: <上司／同僚／会社 のいずれか、なければ「—」>

---

## 📑 各見出しサムネイル

### 1. <章見出し>

- **メインタイトル**: <12〜20字>
- **サブタイトル**: <20〜35字>
- **採用した型**: <①〜⑦>

### 2. <章見出し>

- **メインタイトル**: <12〜20字>
- **サブタイトル**: <20〜35字>
- **採用した型**: <①〜⑦>

（※以降、記事内の全章について同様に繰り返す）
```

### Step 8: X Articles版の生成

note記事をXの「Articles」機能向けに変換し、別ファイルとして保存する。
参考: `SNS運用/post/Xへ記事転載.md` の戦略に準拠。

#### 変換ルール

1. **冒頭フック（タイトル直下）**
   - noteの「みなさん、おはようございます！」挨拶を削除
   - 代わりに**X向けの強いフック**（1〜3文）で始める。ニュースの中で最もインパクトのある事実を冒頭に置き、読者の手を止めさせる
   - **W16学習**: 可能なら1文目に「上司／同僚／会社」の関係性キーワードを仕込む（RT誘発）
   - 例: 「AI業界の「王座」が今日入れ替わった。Claude Opus 4.7がGPT-5-4 Proをぶち抜いた衝撃を5本で解説する。」
   - 例（関係性トリガー強め）: 「あなたの上司が来月"Claude入れろ"と言い出す理由が、今日のニュースで確定した。」

2. **note固有ブロックの削除**
   - `> 📢 AIニュースを追うだけで満足してませんか？` の宣伝ブロックを削除
   - `## 📢 「ニュースを読んで終わり」にしたくない人へ` の有料記事CTAセクションを削除
   - `## 📌 タグ` セクションを削除

3. **本文の調整**
   - markdownの太字（`**`）を削除し、プレーンテキストにする（XのArticlesエディタでは太字が反映されないため）
   - `### 実務観点での感想` → `### 実務で何が変わるか` に見出しを変更
   - `### ○○のポイントまとめ` → `**ポイント**` に簡略化
   - `🔗 **情報源:**` → `🔗 情報源:` に簡略化
   - 各セクションの引用ブロック（`> この章のポイント`）→ 太字の箇条書きに変換
   - 「です・ます」調を「だ・である」調に統一（Xの記事は断定調のほうがエンゲージメントが高い）

4. **末尾CTA（note誘導）**
   - 記事の最後に以下のnote誘導CTAを設置:
   ```
   📝 各ニュースの「自分の仕事にどう活かすか」をさらに深掘りした記事はnoteで毎日更新中。プロンプト集や実践テクニックも公開しています。

   👇 特に反響の大きかった記事はこちら
   🔗 [【PDF付】月80時間の残業→ゼロ。30個のプロンプトで人生が変わった話](https://note.com/chackwill/n/nd015195d2717)

   毎日のAIニュース＋実務で使えるテクニックを発信中。フォローで明日の記事もお届けします👇
   ```

5. **ハッシュタグ（末尾）**
   - X向けのハッシュタグを5つ選定し、記事末尾に1行で記載
   - 例: `#AIニュース #ClaudeOpus4 #OpenAI #生成AI #AI最新情報`

#### 保存先

- `SNS運用/post/article/X-YYYYMMDD.md`（日付は記事の日付）

### Step 9: Xスレッド版の自動生成（`thread`スキル連携）

Step 1〜8 完了後、続けて `thread` スキルの処理を実行する。
これにより、**1コマンドで「note記事リライト → サムネイル文言 → X記事転載版 → Xスレッド版」まで一気通貫**で生成される。

#### 入力データの組み立て

`thread` スキルに以下の3点を渡す（内部処理として実行）：

1. **記事ファイルパス**: Step 1〜6 で保存したリライト後のnote記事ファイルパス
2. **note記事タイトル**: リライト後のh1タイトルを自動取得
3. **note記事URL**:
   - 入力時に2行目でURLが指定されていれば、そのURLを使用
   - 指定がなければ仮URL `https://note.com/chackwill/n/abc` を使用（ユーザーが公開後に手動差し替え）

#### スレッド構成ルール（thread スキル準拠）

- **1投稿目**: 記事全体の導入・フック（興味を引く書き出し）。タイトルのキーワードを含める
- **2投稿目**: note全体への誘導文 ＋ **以下の固定URLを必ず記載**
  ```
  https://note.com/chackwill
  ```
  ※ 当日記事のURLではなく、noteクリエイタートップページのこのURLで固定すること
- **3投稿目以降**: 各項目（今日のニュース5本など）の触りを1投稿ずつ分けて展開
- **最終投稿**: まとめ＋ **当日記事のnote URLを再度記載**（入力で指定がなければ仮URL `https://note.com/chackwill/n/abc`）＋ ハッシュタグ
- 各投稿は **140字以内** に収める（URLは文字数にカウントしない）
- 絵文字は自然に使用（1投稿につき1〜2個まで）
- AIっぽい硬い文章にしない。カジュアルで読みやすいトーンにする
- web制作者であることは記述しない
- `_ kiwami/my-clone/` 内の人格データを反映させる

#### 保存先

- `SNS運用/post/thread/YYYYMMDD_note紹介スレッド.md`（日付は記事の日付）
- フォーマット: 各投稿を `---` で区切り、`## N/M（小見出し）` のヘッダーを付ける

### Step 10: 完了検証と報告（必須）

Step 1〜9 が全て完了した後、**最終応答を返す前に必ず以下を実施**する：

#### 1. 5ファイルの存在検証

Bashツールで以下のコマンドを実行し、5ファイルすべてが存在することを確認（Step 11完了後に最終チェック）：

```bash
ls -la \
  "SNS運用/note/note-YYYYMMDD.md" \
  "SNS運用/title/title-YYYYMMDD.md" \
  "SNS運用/post/article/X-YYYYMMDD.md" \
  "SNS運用/post/thread/YYYYMMDD_note紹介スレッド.md" \
  "SNS運用/action-YYYYMMDD.md"
```

（`YYYYMMDD` は記事の日付に置換）

- ✅ 5ファイルすべて存在 → Step 10-2 へ進む
- ❌ 1つでも欠けている → 該当ステップに戻って再実行する

#### 2. 完了報告

ユーザーへの最終応答に、以下を必ず含める：

- **5ファイルのパス**（クリックできるwikilink形式 `[[...]]` 推奨）
- **各ファイルの要点**（タイトル・ハッシュタグ・投稿数・サムネ文言の採用型・本日アクション件数など）
- **次にユーザーが行うべきアクション**（例: Thread版の仮URL `https://note.com/chackwill/n/abc` を実URLに差し替え、サムネ文言を Fireflyバナーに流し込む、本日アクションプランを順次実行等）

#### 3. スキル一覧の更新確認

本スキル自体（`re-daily.md`）の仕様変更を行った場合のみ、[[Claudian-スキル一覧.md]] を更新する（通常の記事生成タスクでは更新不要）。

### Step 11: 本日アクションプランの生成（必須）

`SNS運用/analytics/` を中心に当日関連ファイルを総ざらいし、ユーザーが**本日１日でこなすべきアクションを時系列で1ファイルにまとめる**。記事生成と並走するX/Threads/プロフ運用の指示が複数ファイルに分散している問題を解決するためのステップ。

#### 11-1. 関連ファイルの読み込み

以下を順に確認し、本日（記事の日付＝今日）に該当する指示を抽出する：

1. **当週の戦略メモ**: `SNS運用/analytics/WNN戦略メモ.md`（今日が含まれる週のISO週番号）
   - 「7日間の運用カレンダー」セクションから本日の予定を抽出
   - 「6つの戦略アクション」「KPI」「測定ポイント」も把握
2. **前週の分析レポート**: `SNS運用/analytics/W(NN-1)分析レポート.md`
   - 「来週のアクション3つ」が今週の指針になっているか確認
3. **統合運用フロー**: `SNS運用/analytics/W*-Claude統合運用フロー.md`（最新のもの）
4. **当日関連ドラフト**:
   - `SNS運用/post/draft/YYYYMMDD_*.md`（連投シリーズ・ピン候補・批評リプ案）
   - `SNS運用/note/_有料記事/*_promo.md`（プロモ並走中なら最新版）
5. **本日のX用日次投稿（必須）**: `SNS運用/post/day*.md` 直下の **本日該当ファイル**
   - ファイル名は通し番号制（`day63.md`, `day64.md` ...）。記事日付に対応する**最新のday番号ファイル**を特定して読み込む
   - 特定方法: `ls -t SNS運用/post/day*.md | head -3` で直近を確認し、当日分（記事日付と一致するもの／未投下のうち最新）を選ぶ
   - 中身からX投稿本文・投下時刻指定・ピン留め指示等を抽出して**タイムラインに必ず含める**
6. **本日のThreads用日次投稿（必須）**: `SNS運用/threads/Threads-day*.md` 直下の **本日該当ファイル**
   - ファイル名規則: `Threads-day{番号}.md`。X側のday番号と対応する同番号ファイルを優先的に選ぶ
   - 特定方法: `ls -t SNS運用/threads/Threads-day*.md | head -3` で直近を確認
   - 中身からThreads投稿本文・連投構成・投下タイミングを抽出して**タイムラインに必ず含める**
7. **テンプレ**: `SNS運用/post/template/*.md`（カレンダー上で参照されているもののみ）
8. **フォロワー改善**: `SNS運用/post/フォロワー改善.md`（CTA・ハッシュタグ統一ルール）

> 不足ファイルがあれば「データなし」と明記し、推測で埋めない。

#### 11-2. 時系列アクションプランの生成

抽出した指示を **時刻順** に並べ替え、以下の項目を**各アクションごとに必ず詳細記述**する：

- **時刻**（例: 7:30 / 12:30 / 21:30 / 夜）
- **媒体**（🐦 X / 🧵 Threads / 📝 note / 🔄 プロフ・ピン操作 / 💬 リプ）
- **アクション名**（例: Day One X案B投下 / 連投シリーズ① 投下）
- **元ファイル**（wikilink形式で参照元を明示）
- **実行手順**（5〜10行で詳細に。下書きの貼付場所、本文コピー元の章番号、投下後にやること等）
- **投稿本文サンプル**（下書きから抜粋。140字なら全文、長文なら冒頭3行）
- **完了チェック**（[ ] 投下完了 / [ ] ピン留め / [ ] リプ最初の1件に返信 など）

#### 11-3. 翌日アクションの追記（簡素）

カレンダーから翌日の予定だけを **3〜6行の箇条書き** で抜き出して末尾に追記する。詳細は翌日の `/re-daily` 実行時に展開する前提でOK。

#### 11-4. 保存先とフォーマット

- **保存先**: `SNS運用/action-YYYYMMDD.md`（vault直下の `SNS運用/` 配下）
- **ファイル名**: `action-` + 記事の日付（8桁、ハイフンなし）
- **完了後**: `SNS運用/archive/action/` に手動で移動される運用（スキル側では移動しない）

#### 11-5. 実行ルール

- **必ずStep 10の検証対象に含める**（5ファイル目）
- 当日のカレンダーが空（休養日等）なら、最低限「本日は投下なし／観察日」と明記しつつ、明日の準備タスク1〜2件を入れる
- **抽象的なアクションは禁止**（「頑張る」「改善する」NG）。必ず実行可能な手順とコピペ本文を含める
- **AIっぽい硬い文章にしない**。チェックリストと短文で「すぐ動ける」状態にする
