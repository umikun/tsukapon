複数の引用RT下書きを「my-clone人格の口調」に書き換え、冒頭に関係性キーワードを差し込んで、規約準拠の最終版として保存するスキルです。

## 入力形式

以下のいずれかの形式で、書き換えたい下書きを受け取ります。

### パターンA: 番号ラベル付きの複数下書き

```
【1番への引用RT下書き】
「〜〜〜〜」
（N字）

【2番への引用RT下書き】
「〜〜〜〜」

【4番への引用RT下書き】
「〜〜〜〜」
```

### パターンB: フリーテキスト（1本のみ）

下書き本文をそのまま貼り付け。自動的に1本として扱う。

### パターンC: 引用元情報付き

```
【引用元URL】https://x.com/xxx/status/123
【下書き】
〜〜〜〜
```

## 処理手順

### Step 1: 人格データと運用ルールの読み込み

以下を必ず読み込む:

1. [[_ kiwami/my-clone/voice/口調パターン.md]] — 語尾・感嘆符・崩し方の基準
2. [[_ kiwami/my-clone/voice/NGワード.md]] — 凍結リスクワードの回避
3. [[CLAUDE.md]] — 「web制作者であることは記述しない」「AIっぽい硬い文章は避ける」などのvault共通ルール
4. [[SNS運用/Threads運用.md]] — 7つの型
5. [[SNS運用/analytics/W17-Claude統合運用フロー.md]]（存在すれば最新のW番号を自動追随）— 関係性キーワード／安全運用ルール

### Step 2: 下書きの解析

- 入力から下書きを N 本切り出す
- 各下書きの「元の意図・訴求ポイント」を把握する
- 文字数・AI臭・属性主張（web制作者／動画編集者／特定の職業）の有無をスキャン

### Step 3: my-clone口調への書き換え

各下書きを以下のルールで書き換える（内容・訴求は維持、語感のみ調整）:

**必ず適用する**
- **です・ます調を基調**にする（感想パートだけ崩してOK）
- **感嘆符（！）を1〜2箇所**に配置
- 「〜ですよね」「〜しちゃう」「〜すぎる」「〜んで」などの**カジュアル崩し**を1〜2回
- **絵文字はワンポイント**で最大1個（🙌💦✨など自然なもの）
- 短文・テンポ感を維持

**必ず除去する**
- 自己属性の主張: 「動画編集やってる身として」「コード触る仕事で」「Web制作の立場から」など
- 男言葉寄りの語尾: 「〜だわ」「〜だぜ」「〜だな」
- AIっぽい定型句: 「いかがでしたか」「活用してみましょう」「〜と言えるでしょう」
- 「確実」「絶対」「100%」「必ず」などの断言誇大系（[[_ kiwami/my-clone/voice/NGワード.md]] 参照）
- 事実不明な自分語り（「うちの会社で」「昨日の商談で」など未確認の体験談）

### Step 4: 関係性キーワードの差し込み

各下書きの**冒頭1文目**に、内容に最もマッチする関係性キーワードを1つ差し込む:

| キーワード | 使いどころ |
|---|---|
| **上司** | 導入判断・予算・経営視点の話題 |
| **会社** | 組織全体・社内ルール・規程の話題 |
| **同僚** | デザイナー・エンジニア・営業等の横の連携話題 |
| **部下** | マネジメント・教育・育成視点の話題 |
| **取引先／お客様** | 営業・CS・契約の話題 |

3本以上の下書きがある場合は、**異なるキーワードを振り分ける**（同じキーワードの連投はspam判定リスクを上げるため）。

### Step 5: 7つの型の判定

書き換え後の下書きが、[[SNS運用/Threads運用.md]] の7つの型のどれに該当するかを判定し、記録する。

- ① 強い冒頭（命令形＋否定）
- ② 具体的な数字
- ③ 常識を覆す
- ④ 対比
- ⑤ 読者の痛み
- ⑥ 今すぐ行動
- ⑦ 損失回避

**2つの型の掛け合わせ**が望ましい（例: ⑦×② / ①×④）。単独型しか当てはまらない場合は、書き換え時にもう1つの型を織り込む。

### Step 6: 文字数のチェック

- 各下書きは **140字以内**（URLは文字数にカウントしない）
- 超過している場合は、**訴求を削らずに冗長な接続詞・副詞を削る**方針で短縮
- 90字未満の場合は、体験談1行か具体数字を1つ足して密度を上げる

### Step 7: 出力ファイルの生成

保存先: `SNS運用/post/draft/YYYYMMDD_quote_draft.md`
（YYYYMMDDは本日の日付。存在すればタイムスタンプ付きで追記 or 上書き確認）

事前に `mkdir -p "SNS運用/post/draft"` でディレクトリを確保。

**ファイルフォーマット**:

```markdown
# YYYYMMDD 引用RT下書き（my-clone口調＋関係性キーワード反映版）

> W○○テーマ「〜〜」／ 冒頭1行に関係性トリガーでRT率目標値狙い
> 運用ルール: [[SNS運用/analytics/W○○-Claude統合運用フロー.md]]
> 投稿前チェック: Claude生成文を自分の言葉で1回書き直す／1日3本以内／手動投稿のみ

---

## 制約事項（絶対遵守）

### 内容の改変ルール
- 元の下書きの**訴求ポイント・数字・固有名詞は変更しない**
- 語尾・語感・トーンのみ調整する
- ただし、事実不明な自分語り（"うちの会社で昨日"等）は削除する

### X規約・安全運用
- **自動投稿は絶対に提案しない**。「手動で投稿ボタンを押す」を明記
- 同じキーワードの連投はspam判定リスクのため、複数本の場合は関係性キーワードを分散
- 1日3本以内の投稿を推奨（出力の注意書きに記載）
- [[Clippings/Post by @kuso_inc on X.md]] のような"大量自動化"系の提案は取り込まない

### 人格の忠実再現
- [[_ kiwami/my-clone/voice/口調パターン.md]] を毎回必ず参照
- web制作者であることは記述しない（CLAUDE.md準拠）
- AIっぽい硬い定型句は除去

### 出力規則
- 保存先は必ず `SNS運用/post/draft/YYYYMMDD_quote_draft.md`
- 同日複数回実行される可能性がある場合は、ファイル内に時刻付きセクションを追記する形で共存
