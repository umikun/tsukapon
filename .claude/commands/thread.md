note記事をX（旧Twitter）のスレッド形式に変換して保存するスキルです。

## 入力形式

以下の3つの情報を改行区切りで受け取ります：

```
記事ファイルパス
note記事タイトル
note記事URL
```

## 処理手順

1. 指定された記事ファイルを読み込む
2. 記事の各項目（見出し・セクション）を把握する
3. 各項目の触り（導入・要点）をX投稿用に140字以内で要約し、スレッド形式に分割する
4. `_ kiwami/my-clone/` 内の人格データを参照し、文体を反映させる

## スレッド構成ルール

- **1投稿目**: 記事全体の導入・フック（興味を引く書き出し）。タイトルを含める
- **2投稿目**: note全体への誘導文 ＋ **以下の固定URLを必ず記載**
  ```
  https://note.com/chackwill
  ```
  ※ 当日記事のURLではなく、noteクリエイタートップページのこのURLで固定すること
- **3投稿目以降**: 各項目の触りを1投稿ずつ分けて展開
- **最終投稿**: まとめ＋ **当日記事のnote URLを記載**（入力で指定がなければ仮URL `https://note.com/chackwill/n/abc` を使用）＋ ハッシュタグ
- 各投稿は140字以内に収める（URLは文字数にカウントしない）
- 絵文字は自然に使用（1投稿につき1〜2個まで）
- AIっぽい硬い文章にしない。カジュアルで読みやすいトーンにする
- web制作者であることは記述しない

## 保存

- 保存先: `SNS運用/post/thread/`
- ファイル名: `YYYYMMDD_スレッド概要.md`（日付は本日の日付を使用）
- フォーマット: 各投稿を `---` で区切り、投稿番号をつける
- **各投稿の本文は必ずコードブロック（``` ```）で囲む**（コピペ用）。字数・補足メモはコードブロックの外に書く

### 出力ファイル例

```markdown
# YYYYMMDD <スレッド概要>

---

## 1/N（フック）

​```
<投稿本文>
​```

> 字数: NN

---

## 2/N（note誘導）

​```
<投稿本文>
https://note.com/chackwill
​```

> 字数: NN（URL除く）

---
```

### チャットへの返答時

保存と同時にチャット内に返す各投稿本文も、**1投稿につき1コードブロック**で出力すること。説明文（採用した型・字数など）はコードブロックの外。

## 入力データ

$ARGUMENTS
