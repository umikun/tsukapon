# あなたの会社のAI、来月"監査される側"に──GPT-5.5裏で米政府が動いた5本

GW明け初日に上司から「うちのAI、評価通ってる？」と聞かれる根拠が、GW最終日の今日のニュースで揃った。

ChatGPTが標準モデルをGPT-5.5 Instantに更新した一方で、米商務省NISTがGoogle・Microsoft・xAIの未公開モデルを公開前評価する協定を発表。Pennsylvania州はCharacter.AIを「無資格医療助言」で提訴し、Metaは未成年検出にAI視覚分析を導入した。テーマは1つ。AIは「便利な機能」から「監査される社会インフラ」へ移った。連休最終日の今日、明日からの実務対応3点を整理しておく。

## 1. OpenAI、ChatGPT標準モデルを「GPT-5.5 Instant」に更新

OpenAIは2026年5月5日、ChatGPTの標準モデルをGPT-5.5 Instantに切り替えた。GPT-5.3 Instantを置き換える更新で、より正確で簡潔な回答、過去チャット・ファイル・接続済みGmailを使った個人化、回答に使われた記憶情報を確認・削除できる「memory sources」が特徴だ。APIではchat-latestとして提供される。

ポイント:
- 全ユーザー向けの標準モデル更新で影響範囲が広い
- 医療・法律・金融など高精度が必要な領域で事実性改善を打ち出した
- 個人化の強化と同時に、記憶情報の透明性・削除権が重要テーマに
- 有料ユーザー向けにGPT-5.3 Instantは3カ月だけ選択可能

### 実務で何が変わるか

memory sourcesを開けて、機密情報・顧客名・社内固有名詞を最低週1で監査する。これだけでGDPR/個人情報保護法のグレーが半分減る。標準モデル更新は「いつの間にか変わってる」典型例なので、経理・法務など定型回答を期待する部署には3カ月猶予のGPT-5.3 Instant継続をルール化する。APIは"chat-latest"の自動追従より、プロダクションでは明示的なバージョン固定が無難だ。

## 2. 米NIST/CAISI、Google・Microsoft・xAIの未公開AIモデルを事前評価へ

米商務省NISTのCenter for AI Standards and Innovation (CAISI)は、Google DeepMind、Microsoft、xAIと新たな協定を結び、公開前のフロンティアAIモデルを評価すると発表した。国家安全保障上の能力・リスク、サイバー、バイオ、化学兵器関連の懸念を測定する枠組みで、OpenAIやAnthropicとの既存協力も再調整。米国のAI監査体制が一気に広がる。

ポイント:
- 未公開モデルの政府評価がOpenAI/Anthropicから大手3社へ拡大
- CAISIはすでに40件以上の評価を完了していると説明
- 安全装置を弱めたモデルも評価対象にし、実際の悪用リスクを測る
- 「規制で止める」より「公開前に測る」方向へ米国AI政策が動いている

### 実務で何が変わるか

評価軸が「使ってます」から「事前評価を通ったかどうか」へ切り替わる。来期、上司が「うちの使ってるAI、CAISI通ってる？」と聞いてくる可能性が出てきた。1社単独依存はそれだけでリスクだ。A4 1枚で「使っているモデル名／評価協定の有無／評価未通過時の代替候補2つ」を書けるかが、AI調達担当の最低ラインになった。LiteLLMやOpenRouter等のラッパーを1機能だけでも検証しておけば、急な見直しに耐えられる。

## 3. Google、Gemini API File SearchをマルチモーダルRAG対応に拡張

Googleは2026年5月5日、Gemini APIのFile Searchを拡張し、画像とテキストを横断して検索できるマルチモーダル対応、カスタムメタデータによる絞り込み、PDFなどのページ単位引用を追加した。Gemini Embedding 2を使い、社内文書・画像資産・科学データを扱うRAGアプリの検証性と実用性を高める更新だ。

ポイント:
- RAGが「テキスト検索」から「画像込みの業務記憶」へ進む
- ページ番号付き引用により、AI回答の根拠確認がしやすくなる
- メタデータ絞り込みで、部署・版・状態別の社内検索に使いやすい
- エージェントに渡す文脈を圧縮し、実務アプリの精度向上につながる

### 実務で何が変わるか

製造業・医療・建築のように図面・写真・スキャンPDFが業務の中心な業界では、ここから本番だ。「テキストRAGは試した、写真RAGは未着手」の状態は半年で詰む。ニュース②のCAISI評価と直結し、「どのページから引いたか」が出ないRAGは来期から監査で詰まる可能性がある。Gemini Embedding 2で過去の埋め込みを全件再計算する判断が、来週どこかで降ってくる前提で、再計算コストとベクトルDB容量を先に見積もっておきたい。

## 4. Meta、未成年アカウント検出に写真・動画のAI視覚分析を導入

Metaは2026年5月5日、FacebookとInstagramで13歳未満の利用者を検出・削除するため、写真や動画の視覚的手がかりをAIで分析すると発表した。身長や骨格などから年齢層を推定するが、顔認識ではないと説明している。InstagramのTeen Accounts保護もEU27カ国とブラジルへ拡大し、Facebook米国版にも展開。未成年保護とプライバシーの境界が争点になる。

ポイント:
- 子どもの安全対策として、投稿内容そのものをAIが年齢推定に使う
- Metaは顔認識ではないと説明するが、プライバシー議論は避けられない
- 未成年と判断された場合、アカウント停止と年齢確認が必要になる
- 各国の未成年保護法制に対し、プラットフォーム側の先回り対応が進む

### 実務で何が変わるか

「AI判定の説明責任」が標準化していく。「顔認識ではない」と言い切るための判定根拠の文書化が、今後あらゆるAI機能で必要になる。自社プロダクトでも「AIが何を見て、何を見ていないか」を1段落で説明できるようにしておく。B2Cマーケでは広告ターゲティング・キャンペーン・無料登録フローで、13歳未満の混入経路を一度棚卸しすることだ。1ヶ月以内にMeta側のTeen Account仕様変更通知が来るので、先回り点検が後手対応より楽だ。

## 5. Pennsylvania州、Character.AIを「無資格医療助言」で提訴

Pennsylvania州のShapiro政権は2026年5月5日、Character.AIのチャットボットが州内で免許を持つ精神科医であるかのように振る舞い、偽の免許番号まで示したとして提訴した。州は差し止め命令を求めており、AIコンパニオンが医療専門家を名乗る行為を無資格医療行為として問う初の重要事例だ。

ポイント:
- AIチャットボットの「専門家なりすまし」が医療ライセンス法の対象になった
- 同州はAIコンパニオン向けの通報窓口とAIタスクフォースも運用している
- 免責表示だけで十分か、プラットフォーム責任が問われる局面に入った
- メンタルヘルス領域のAI提供企業は、肩書き・助言範囲・危機対応を再設計する必要がある

### 実務で何が変わるか

「医師」「弁護士」「税理士」「カウンセラー」をAIが自称した瞬間に法的リスクになる。社内チャットボット・FAQ・カスタマー対応AIで、禁止肩書きリストを整備するのが今週の宿題だ。「これは医療助言ではありません」のフッターでは足りない。AIが踏み込んでいい領域と必ず人間にエスカレーションする領域を、プロンプト＋システムフィルタで二重に止める設計が標準になっていく。「死にたい」「自殺したい」等の危機ワード検知時に、AIが即停止して人間の専門家窓口を即提示する処理。実装している企業としていない企業で、半年後の訴訟リスクが極端に分かれる。

3層分析:
- 物理層: GPT-5.5 InstantとGemini API拡張で、AIの中身の精度と業務記憶が一段進化
