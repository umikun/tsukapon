## Gemini CLIって何？

Gemini CLIは、Googleが2025年6月にリリースしたオープンソースのAIエージェントです。Apache 2.0ライセンスで公開されてて、ターミナルからGeminiモデルの力をフルに使えるツールですね。

**「ターミナルに住み着くAIの相棒」**みたいなもんです。

コードの生成、ファイル操作、シェルコマンドの実行、さらにはGoogle検索でリアルタイム情報を引っ張ってくるところまで、全部ターミナルの中で完結しちゃう！しかもこれ、**個人のGoogleアカウントがあれば無料で使える**っていうのが一番のインパクトですよね！

## 何がすごいのか、ポイントを整理

### 1. 無料枠がめちゃくちゃ太い

Gemini CLIの無料枠は**60リクエスト/分、1日1,000リクエスト**です。個人のGoogleアカウントでログインするだけで、Gemini 3モデルにアクセスできちゃう。

これ、Claude Codeを使うにはProプラン以上が必要なことを考えると、かなりハードル低いですよね。「ちょっとターミナルAIエージェントを試してみたい」って人にとっては最高の入り口だと思います。

正直に言うと、僕みたいに普段Claude Code使ってる人間からすると「無料でここまで出来ちゃうの？」ってちょっと焦るレベルです💦

### 2. 100万トークンのコンテキストウィンドウ

**100万トークンのコンテキストウィンドウ**。これがデカい。

大規模なモノレポとか、ファイルがめちゃくちゃ多いプロジェクトでも、一度にドカッとコードベース全体を読み込めちゃうんですよね！コンテキストが切れて「さっきの話忘れちゃった」みたいなことが起きにくい。

Web制作でも「このプロジェクト、ファイル多すぎてAIに全体像つかんでもらうの大変だな…」って思うことあるんで、これはめっちゃありがたい💦

### 3. Plan Mode（計画モード）が登場

2026年3月に入ってからの目玉アップデートが**Plan Mode**です！

Gemini CLIが「まず全体を分析→計画を立てる→それから実行」っていう二段階アプローチを取れるようになった機能ですね！Plan Mode中は**読み取り専用**なんで、うっかりファイルを書き換えちゃう心配がない。コードベースを調べて、パターンを見つけて、ドキュメントを読んで、計画を立てる。ここまでが安全にできるんです！

Claude Codeにも似た仕組みはあるんですが、Gemini CLIのPlan Modeは**リサーチ用サブエージェント**が組み込まれてたり、計画にフィードバックを付けられるアノテーション機能があったり。かなり本格的です！

さらに面白いのは**`ask_user`ツール**が組み込まれてて、計画中に「ここ、こういう方針でいいですか？」ってAIの方から質問してくれるんですよね。ゴールの認識ズレを計画段階で修正できるんで、実行してから「思ってたのと違う…」ってなるリスクがグッと減ります！

しかも計画段階ではGemini 3.1 Proっていう推論性能の高いモデルが使われるんで、アーキテクチャレベルの判断も期待できますね💦

### 4. ビルトインツールが充実

最初からいろんなツールが使えるのも嬉しいポイントです。

- **ファイル操作**: 読み書き、検索、リファクタリング
- **シェルコマンド実行**: ターミナルコマンドをAIが代わりに叩いてくれる
- **Google Search grounding**: リアルタイムのWeb情報を使って回答の根拠を補強
- **Webフェッチ**: URLからコンテンツを取得
- **Interactive Shell**: `vim`や`git rebase -i`みたいな対話型コマンドもGemini CLI内で直接実行できる

特にGoogle Search groundingは「Google製」だからこその強みですよね。AIの回答に最新のWeb情報を混ぜ込んでくれるんで、古い情報をドヤ顔で出してくるリスクが減ります。

あとInteractive Shell対応が地味にすごくて、ターミナルの色付き出力もちゃんとレンダリングしてくれるし、AIとの双方向コミュニケーションを維持したまま`top`コマンドとかも実行できちゃう。**ワークフロー全体をGemini CLIの中で完結させられる**設計になってるんですよね！

### 5. MCP対応でツール拡張も自由自在

Gemini CLIは**MCP（Model Context Protocol）**に対応してます。MCPっていうのは、AIモデルと外部ツールを繋ぐための共通規格で、最近Claude CodeやCopilotでも対応が進んでるやつですね。

これがあると、GitHubのPRを操作したり、Slackにメッセージ送ったり、データベースにクエリ投げたりっていう外部連携がプラグイン感覚でできるようになります。

## インストールはびっくりするほど簡単

セットアップの手軽さもGemini CLIの魅力です。Node.jsが入ってれば、コマンド一発で試せます。

```bash
# すぐに試すならこれだけ！
npx @google/gemini-cli

# ちゃんとインストールするなら
npm install -g @google/gemini-cli

# macOS/LinuxならHomebrewでも
brew install gemini-cli
```

起動したらGoogleアカウントでログインするだけ。APIキーの管理とか面倒なことは一切なし。

あとは `gemini` ってターミナルで打つだけで、インタラクティブモードが立ち上がります。プロジェクトのディレクトリで起動すれば、勝手にコードベースを認識してくれるんで、すぐに「このバグ直して」とか「テスト書いて」とか指示が出せます。

## GEMINI.mdでプロジェクトごとにカスタマイズ

プロジェクトのルートに**GEMINI.md**っていうファイルを置くと、Gemini CLIの挙動をカスタマイズできます。Claude Codeでいう`CLAUDE.md`と同じ発想ですね。

プロジェクトのコーディング規約とか、使ってるフレームワークの情報とか、「このディレクトリは触らないで」みたいな指示を書いておくと、AIがそれを読んで空気を読んでくれるようになります。

僕はViteでのWeb制作がメインなんで、こういうカスタマイズファイルで「scssのコンパイルはこうやってる」「このプロジェクトはWordPressのテーマだよ」みたいに教えておけるのはめっちゃ助かります。

## 知っておくと便利な追加機能

記事を書くにあたって調べてたら、けっこう便利な機能がまだあったんで追記しておきますね！

### チェックポイント機能

作業の途中経過を**チェックポイント**として保存できる機能。「ここまでの状態を保存して、明日続きからやりたい」みたいな使い方ができます。長期的なプロジェクトでセッションをまたいで作業するときにめっちゃ便利！

### マルチモーダル対応

Gemini CLIは**画像やPDFも入力として受け取れる**んです。たとえばデザインのスクリーンショットやワイヤーフレームのスケッチを渡して「これをコードにして」みたいな指示ができちゃう。Web制作の現場だと、デザイナーからもらったカンプをそのまま渡せるのはかなりアツいですね💦

### ヘッドレスモード

CI/CDパイプラインやスクリプトの中でGemini CLIを自動化するための**ヘッドレスモード**もあります。対話的な操作なしで、コマンドラインから直接タスクを実行できるんで、定期的なコードレビューとかドキュメント生成の自動化に使えます！

### VS Codeとの連携

実はGemini CLIの機能の一部は、VS Codeの**Gemini Code Assist**拡張機能の中でもそのまま使えるんです！MCP接続、`/memory`や`/stats`コマンド、Web検索、ファイル操作とか。ターミナルが苦手な人でもVS Codeの中からGemini CLIの力を借りられるのは嬉しいですよね！

## 正直なところ、Claude Codeとどっちがいいの？

ここが一番気になるところだと思うんで、僕なりの感想を書いておきます。

**Claude Codeが強いところ：**

- コードの推論力と分析の深さ。複雑なリファクタリングとかアーキテクチャの相談はClaude Codeのほうが的確な印象
- マルチファイル編集の精度。複数ファイルにまたがる変更を一発で正確にやってくれる
- MCP連携のエコシステムが成熟してる（300以上のインテグレーション）
- SWE-bench（自律的なバグ修正のベンチマーク）のスコアはClaude Codeが上

**Gemini CLIが強いところ：**

- とにかく無料で始められるハードルの低さ
- 100万トークンの巨大コンテキストウィンドウ
- Google Search groundingによる最新情報へのアクセス
- Plan Modeの読み取り専用アプローチが安心感ある
- Googleエコシステムとの親和性

個人的には「メインの開発作業はClaude Code、ちょっとした調査やサブのプロジェクトはGemini CLI」みたいな使い分けがいいんじゃないかなと思ってます。

特にWeb制作の現場だと、新しいライブラリの使い方を調べながらコード書くみたいな場面が多いんで、Google Search groundingが使えるGemini CLIは「調べもの＋コーディング」の一気通貫がやりやすいんですよね。

ただ、自律的にバグを見つけて直すとか、大規模なリファクタリングを任せるとかってなると、やっぱりClaude Codeの推論力には一日の長がある感じです。SWE-benchのスコア差（Claude Opus 4.6: 72〜77% vs Gemini 3 Pro: 63〜67%）は縮まってきてるものの、まだClaude側がリードしてる状況ですね。ただGemini 3.1 ProがPlan Modeに入ったことで、今後のアップデートでさらに差が縮まる可能性は全然あります！
