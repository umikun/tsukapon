## 何が起きたのか

2026年3月31日、Anthropicが開発するAIコーディングツール「Claude Code」の全ソースコードが、npmパッケージの設定ミスにより誤って公開されました。

`@anthropic-ai/claude-code` パッケージのバージョン2.1.88に、59.8MBのJavaScriptソースマップファイル（cli.js.map）が含まれた状態でnpmレジストリに公開されてしまったのです。このソースマップには、難読化されていない完全なTypeScriptソースコードがそのまま含まれていました。

## 発覚の経緯

セキュリティ研究者のChaofan Shou氏が、米国東部時間3月31日午前4時23分頃にX（旧Twitter）上でこの漏洩を公表しました。

それ以前から開発者たちは12MBに圧縮されたcli.jsファイルのリバースエンジニアリングを試みていましたが、今回の事故により未難読化の完全なソースコードが一気に露出する結果となりました。

## 漏洩の原因

原因はシンプルな設定ミスでした。

- Claude Codeが使用するBunランタイムは、デフォルトでソースマップを生成する
- `.npmignore` に.mapファイルが追加されていなかった
- そのため、ソースマップがそのままnpmパッケージに含まれて公開された

Anthropicはこれを「人為的ミスによるリリースパッケージングの問題であり、セキュリティ侵害ではない」と説明しています。

## 漏洩の規模

流出したコードベースの規模は以下の通りです。

- TypeScriptファイル数：約1,884〜1,906ファイル
- コード行数：約512,000行
- 非圧縮サイズ：59.8MB

### 露出した内容

- Claude Codeのエージェントハーネス全体のアーキテクチャ
- 約40の権限ゲート付きツールシステム（ファイル読み取り、bash実行、Webフェッチ、LSP統合など）
- ベースツール定義：29,000行のTypeScript
- サブエージェントを生成するマルチエージェント「スワーム」システム
- JWT認証付きIDEブリッジ
- 2,500行以上のbashバリデーションロジック
- 階層化メモリ構造
- トークンカウント、リトライロジック、ストリーミングレスポンス処理
- パーミッションモデルとアクセス制御システム
- 44のフィーチャーフラグ（うち20は未出荷機能用）

## 発見された未リリース機能

コード解析の結果、3つの大きな未発表機能が見つかりました。

### 1. BUDDY（ターミナルペットシステム）

18種類の生物（カピバラ含む）から選べるターミナルペット機能。手続き的に生成される個性と、DEBUGGING・PATIENCE・CHAOSといったステータスを持ちます。4月1〜7日のサプライズローンチ、5月に正式リリースが予定されていた模様です。

### 2. KAIROS（常駐バックグラウンドデーモン）

ファイルの変更を監視し、イベントをログに記録し、アイドル時にメモリ統合を実行する常駐プロセス。ファイルの直接プッシュ、プッシュ通知、プルリクエスト監視などの専用ツールを含みます。

### 3. ULTRAPLAN（リモートマルチエージェント計画）

10〜30分のリモートマルチエージェント計画セッションをスケジュールし、複雑な計画をクラウドコンテナ上のClaude Opus 4.6にオフロードする機能。

## コミュニティの反応と拡散

Anthropicがnpmパッケージを迅速に削除したものの、拡散は止められませんでした。

- 数時間以内に複数のGitHubリポジトリにソースコード全体がミラーリングされた
- 最も急成長したリポジトリは、わずか2時間で30,000スター・40,200以上のフォークを獲得
- 発表ポストは約1,000万ビュー・1,500コメントを記録

コミュニティの反応は賛否両論でした。技術アーキテクチャを理解できることを歓迎する開発者がいる一方、「ソースが公開されたからといってオープンソースになったわけではない。ソースコードのコピーや再配布はライセンス違反である」という法的警告も上がっています。

## Anthropicの対応

- インシデントを確認し、「人為的ミスによるリリースパッケージングの問題」と説明
- 「顧客の機密データや認証情報は一切関与・露出していない」と明言
- モデルの重みや推論インフラは影響を受けていないと明確化
- 影響を受けたnpmパッケージをレジストリから迅速に削除

なお、これは2025年2月に未リリースのモデル仕様が漏洩した件に続き、数ヶ月で2度目の重大インシデントとなりました。

## セキュリティ・戦略上の影響

今回の漏洩は競争上重大な影響を持ちます。

- 競合他社がClaude Codeのアーキテクチャパターンを研究し、少ないR&Dコストで同等のエージェントを構築できる可能性
- セキュリティガードレールの特定と悪用の手がかりを研究者に提供
- HooksやMCP（Model Context Protocol）サーバーのオーケストレーションロジックが完全に露出
- Anthropicの製品ロードマップと未出荷機能が公開
- 内部の命名規則や開発優先度が明らかに

## 教訓

この事件はソフトウェアサプライチェーンセキュリティにおける重要な教訓を残しました。

- ビルドツールのデフォルト設定を盲信しない（Bunはデフォルトでソースマップを生成する）
- `.npmignore` や `.gitignore` の設定を必ずレビューする
- CI/CDパイプラインにパッケージ内容の検証ステップを組み込む
- 公開パッケージのリリース前に、含まれるファイルの監査を徹底する

たった一つの設定ファイルの見落としが、50万行のソースコードと未発表の製品ロードマップを世界中に公開してしまう結果となった、象徴的な事例です。
