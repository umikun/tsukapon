## 1. OpenAI、GPT-Realtime-2など3つの音声APIモデルを発表

通話中の音声AIが、ただ返事するだけじゃなく、考えて動くフェーズに入った。

OpenAIは5月7日、API向けにGPT-Realtime-2、GPT-Realtime-Translate、GPT-Realtime-Whisperの3モデルを発表した。GPT-Realtime-2はGPT-5級の推論・128Kコンテキスト・並列ツール呼び出しに対応。会話中の割り込みや訂正、ツール実行、長文脈の保持が一体になった「**実行する音声AI**」が動き出した。

### 実務で何が変わるか

評価軸が「テキストチャットボットを置けたか」から「**通話中に判断して動く音声エージェントを置けたか**」に切り替わる。来月の経営会議で「うちのコールセンターもAI化」と振られたら、テキスト前提ではなく音声前提で答えられるかが試される。GPT-Realtime-2はGPT-5級推論で並列ツール呼び出しを持つので、「電話で問い合わせ → DBを参照 → 翻訳 → 回答」が1呼び出しで回る。コールセンター・旅行・医療予約・営業同行の業務棚卸しを、テキスト前提から音声前提に組み直すタイミングだ。

## 2. AWS、Bedrock AgentCore Paymentsをプレビュー公開

AIエージェントが会社のクレカで勝手にAPIを買う時代が、今日プレビューで来た。

AWSは5月7日、Amazon Bedrock AgentCore Paymentsをプレビュー公開した。CoinbaseとStripeのウォレット基盤を使い、AIエージェントがWebコンテンツ・API・MCPサーバー・他のエージェントへ即時マイクロペイメントできる。**利用者の明示許可・セッション単位の支出上限・ログ／トレース**を備え、エージェント経済の決済インフラをクラウド側から押さえる動きだ。

### 実務で何が変わるか

ソフトウェアの課金単位が「人の月額契約」から「**エージェントの実行ごと**」に変わる。経理から「AI関連の請求書、誰がいくらまで承認できる？」と振られた瞬間に答えられないと、現場でAIエージェントを動かせなくなる。**支払い権限マトリクス**を社内で先に作っておくと、来月のAI調達会議で一段優位に立てる。CoinbaseとStripeが初期パートナーで、x402プロトコルが標準寄りになる流れも要チェック。

## 3. Anthropic、SpaceXのColossus 1計算資源を確保

Claude Codeが「混んでてレート制限」と返してくる原因は、モデルの賢さじゃなく電力とGPU不足だった。今日それが裏取りされた。

AnthropicはSpaceXのColossus 1データセンターの全計算能力を利用する契約を結んだと報じられた。**300MW超・22万基超のNVIDIA GPU相当**の新容量が1カ月以内に使える見込みで、Claude Pro/MaxやClaude Code、Opus APIの制限緩和に直結する。AI競争はモデル性能だけでなく、即座に使える電力・GPU・データセンター在庫を取れるかに移った。

### 実務で何が変わるか

AIサービスの品質差が「**混雑時に計算力を出せるか**」で決まる時代に入った。同じClaude Maxでも、ピークタイムに詰まらないのは「契約しているSaaSの裏側で何MWの計算資源が確保されているか」次第。AI予算の組み方を「モデル料金100%」から「モデル料金70% + 品質低下時のフォールバック確保30%」に切り替える発想が必要になる。Anthropic / OpenAI / Google の3社が、それぞれどこから電力・GPUを取っているかを社内で1ページに整理しておくと、調達リスクの説明が一段速くなる。

### 📋 すぐ試せるアクション

- [ ] **3社の計算資源戦略を1ページにまとめる**：Anthropic / OpenAI / Google が誰からGPUと電力を取っているか
- [ ] **主要AIサービスのSLA確認**：「混雑時に速度落ちる契約か／レート制限が走る契約か」を読み直す
- [ ] **AI予算 7:3 割りの提案**：モデル料金70% / フォールバック確保30% で予算を設計し直す

### ポイント

- Colossus 1はH100、H200、GB200を含む大規模AI計算基盤
- Anthropicは需要増に対し、AWS／Google／Microsoft／NVIDIA系契約だけでは即時容量が足りない局面に
- Claude CodeやOpus APIのレート制限緩和は計算資源確保と直結
- AIサービス品質差は「モデルの賢さ」ではなく「ピーク時の計算力」で決まる

## 4. EU、AI Actを簡素化し高リスクAI期限を延期

「EU AI法が緩んだ」じゃない。「**実装猶予が2年伸びた**」が正しい読み筋だ。

欧州委員会は5月7日、欧州議会とEU理事会がAI Actを簡素化する政治合意に達したと発表した。**雇用・教育・重要インフラ・移民／国境管理**などの高リスクAIは2027年12月2日、リフトや玩具など製品組込AIは2028年8月2日から適用される見通し。さらに「nudification」（非同意の性的画像生成）アプリ規制も盛り込まれ、競争力と安全性の再調整が進む。

### 実務で何が変わるか

EU市場向けAIサービスを持つ企業にとって、これは「**安心」じゃなく「準備期間が来た」**シグナルだ。2027年12月の高リスクAI適用日から逆算すると、文書化・リスク評価・監査体制の整備に**最低18ヶ月**は必要。猶予期間を「対応を後回しにする時間」に使うのではなく、「標準化と監査ツールが揃うのを待ちつつ、社内の文書化を始める時間」に使うかで、来年の競争力に大きく差がつく。来月の取締役会で「EU AI法の対応どうなった？」と振られた時、「猶予出ました」だけで済ませない準備を今日始めるべきだ。

### 📋 すぐ試せるアクション

- [ ] **EU向けサービスの棚卸し**：自社のAI利用シーンのうち、雇用／教育／重要インフラ／移民管理に該当するものをリスト化
- [ ] **適用日のカレンダー登録**：2027年12月2日（高リスクAI）／2028年8月2日（製品組込AI）の6ヶ月前から逆算
- [ ] **nudification規制の対象範囲確認**：自社プロダクトに該当機能がないか、API利用規約レベルでチェック

### ポイント

- 高リスクAIの主要義務は当初想定より後ろ倒し、標準や支援ツール整備を待つ
- 製品組込AIはさらに遅い2028年8月2日適用予定
- 非同意の性的画像生成など被害が明確な用途への規制は強化方向
- 「延期されたから安心」ではなく「実装準備期間を得た」と読む

## 5. OpenAI、ChatGPT広告パイロットを日本などに拡大へ

「会話の中で広告に出会う」市場が、来月から日本でテスト開始する。SEO担当はもうウォッチに入るタイミングだ。

OpenAIは5月7日、ChatGPT内広告パイロットを今後数週間で英国・メキシコ・ブラジル・**日本・韓国**へ拡大すると発表した。無料・低価格層を支える収益源として広告を試す一方、回答への影響はないこと、広告は明確に表示すること、会話内容を広告主に共有しないことを強調している。日本の広告主にとって、検索広告とは別の「**会話内広告**」市場が始まる合図だ。

### 実務で何が変わるか

SEOと広告運用の前提が「検索結果で見つかる」から「**AIとの相談中に出会う**」へ広がる。今月から月1回、自社競合キーワードでChatGPTに質問する習慣を作るかどうかで、半年後の流入チャネル設計に差が出る。最低限やるのは、①競合キーワードでの ChatGPT 検索結果を月1で取得、②自社ブランド名で質問した時の回答内容を保存、③ ChatGPT が引用してくる一次情報がどこかを把握、の3つ。検索エンジンSEOの上に、「AIに引用されるための一次情報整備」がレイヤーとして乗る形になる。

### 📋 すぐ試せるアクション

- [ ] **競合キーワード×3でChatGPT検索結果を取得**：1ページに保存して月1回更新
- [ ] **ブランド名でAI回答に出るかを確認**：自社サイトの一次情報整備、コミュニティ言及の獲得を再評価
- [ ] **OpenAI広告日本展開のフォロー先設定**：OpenAI公式ブログの日本リリースをRSSやAlertでウォッチ

### ChatGPT広告対策チェックリスト（コピペ用）

```
1. 自社の主要競合キーワードでChatGPTに質問してみる

4. 自社サイトがChatGPTの引用元として使われているか
5. 取得結果を社内共有（月1回）
```

- SEOは「検索結果順位」から「AI回答内での引用率」が新軸に
