## 1. MicrosoftとOpenAIの「独占」終了──"AI調達ポートフォリオ"を3行で握る

> **この章のポイント**
> - MicrosoftとOpenAIの独占的パートナーシップが正式に修正、OpenAIが他社クラウド（Google・Amazon等）に技術提供可能に
> - Microsoftは主要パートナーの座を維持しつつ収益分配率を変更、2032年までの知財利用権だけ確保
> - 背景にはMicrosoftのOpenAI依存への投資家懸念と、自社モデル「MAI-1」開発加速

### 概要

MicrosoftとOpenAIは、これまでの独占的パートナーシップを修正し、OpenAIが他社のクラウドプラットフォーム（Google CloudやAWS等）へ技術提供することを容認する新たな契約を締結しました。Microsoftは今後も主要なクラウドパートナーであり続け、**2032年までの知財利用権**を維持しますが、収益分配（レベニューシェア）の枠組みも変更されます。背景にはMicrosoftのOpenAI依存に対する投資家の懸念と、自社モデル「MAI-1」開発への注力があります。「最強タッグ」から「戦略的距離感」への移行は業界全体の勢力図を塗り替える可能性があり、Azure OpenAIに寄せていた会社のAI調達ポートフォリオがGW明けに再点検対象になるフェーズに突入しました。

### 実務で何が変わるか

煽り版の見出しはこう書かれます: 「MS-OpenAI離婚！クラウド戦争の幕が開いた！」
実態はこう: 離婚じゃなくて**"複婚解禁"**。OpenAIがGCP・AWSにも乗ったということは、"OpenAI=Azure"で固定して見積取らずに更新してきた契約が、GW明けに「他社経由なら何%安いの？」を詰められるフェーズに入っただけ。

これ、「うちはAzure OpenAI契約してるから関係ない」で流すと、来期の予算策定で「Azure経由とGCP経由とAWS Bedrock経由でOpenAIモデルの料金比較を出して」と経営から振られて、社内に誰も比較表を持っていない1本です。昨日(4/29)書いた「MS-OpenAI収益分配削減」が"独占解消"まで踏み込んだ続報という位置付け。

核心は**同じOpenAIモデルが3クラウドから調達可能になった**こと。これまで「OpenAI使うならAzure一択」で思考停止できましたが、今後はGPT-5系・Codex系・Sora系の各モデルごとに「Azure / GCP / AWS Bedrock のうちどこ経由が一番安いか／速いか／セキュリティ要件に合うか」を選び直すフェーズに入ります。Azure包括契約で全部寄せている会社は、来期に「同じ機能で月数十万〜数百万のコスト差」を見過ごし続けることになります。

ただし、3クラウド全部を同時運用する必要はない。「メイン1つ＋セカンダリ1つ、機能単位で見直す」3行ルールだけ握れば、調達ポートフォリオを3ヶ月で組み替えられます。

実務で効く3つの視点。

**1つ目、AI調達ポートフォリオをA4 1枚で作る**。列に①利用機能(Chat／要約／画像生成／コード生成／RAG)、②現在のクラウド経由(Azure / GCP / AWS / 直接)、③同モデルの他クラウド見積、④切替工数(契約・データ移転・社内承認)を書く。「うちのAzure包括契約、機能単位でバラすと月いくら浮くか」が経営会議で1秒で示せます。

**2つ目、見積取得依頼を四半期1回テンプレ化**。新契約・更新時に「OpenAIモデル○○の料金を、Azure・GCP・AWS Bedrock で比較見積取得」を購買フローに組み込む。法務・購買・情シスで30分の見積取得ルーチンを四半期1回回せば、「気づいたら他社が30%安くなっていた」を防げます。

**3つ目、包括契約のロックイン条項を法務に再確認**。Azure ESA・OpenAI Enterprise契約に「契約期間中の他社並行利用禁止」「データの他社移転制限」が入っていないか、契約書を1枚法務に渡して2週間で回答もらう。ロックインがあれば次回更新時の交渉材料、なければGW明けに即動ける状態を作れます。

## 2. 米国防総省（DoD）がGoogle Geminiを機密ワークロード本格採用──"マルチベンダー人質回避ルール"

> **この章のポイント**
> - 米国防総省が機密性の高い軍事業務でGoogle Gemini AIの利用を大幅拡大
> - DoDのAI担当責任者キャメロン・スタンレー氏が「マルチベンダー戦略」を明言、Anthropicへの依存を意図的に減らす方針
> - Gemini導入で週単位で「数千時間の工数削減」を実現、AIが国家安全保障の実戦ツール化

### 概要

米国防総省は、機密性の高い軍事業務においてGoogle Gemini AIの利用を大幅に拡大することを認めました。DoDのAI担当責任者キャメロン・スタンレー氏は、特定のベンダーへの過度な依存を避ける**マルチベンダー戦略**の一環として、Anthropicへの依存を減らし、GoogleやOpenAIとの連携を強化していると説明しています。Geminiの導入により、週単位で数千時間の工数削減が実現しているとされ、AIが国家安全保障の最前線で「実戦レベル」の効率化ツールとして不可欠な存在になったことを示しています。"Anthropic一択"で固めた業務が、来期に同じ理由で揺さぶられる前夜に来ました。

### 実務で何が変わるか

煽り版の見出しはこう書かれます: 「米軍がClaudeを切り捨てた！Anthropic終了！」
実態はこう: 米軍は"Anthropic切り捨て"じゃなく**"単一ベンダー人質回避"の設計**に動いただけ。同じ理屈で来期、自社の経営も「Claude一択」「GPT一択」の業務に対して「なぜ複数ベンダー検討してないのか」を議事録レベルで詰めてきます。

これ、「うちは軍事関係ない」で流すと、来年の取引先監査で「主要AIベンダー1社停止時のBCPは？」を問われて、社内に誰も答えを持っていない1本です。昨日(4/29)書いた「中国Manus買収阻止」と同じ構造──AIベンダー選定が"国家戦略レベル"で評価されるフェーズに入りました。

核心は**マルチベンダー戦略＝単一ベンダー人質回避コスト**ということ。米軍が動いたということは、同じロジックで来年、金融・医療・行政・大手SaaSが「主要業務はAIベンダー2社並列」を契約条件に入れてくる流れが見えています。今のうちに「主要1業務だけ並列構成」を作っておけば、取引先監査の質問で固まる側から、1秒で答えられる側に回れます。

ただし、全業務を2社並列にする必要はない。「機密度が高い／業務継続が必須／替えが効かない」3条件のいずれかに当てはまる業務だけ、2社並列にするだけで十分です。

実務で効く3つの視点。

**1つ目、人質リスク3条件で業務を選別**。①機密度（顧客データ・契約情報・人事情報を扱うか）、②業務継続性（停止時に売上が止まるか）、③代替可能性（他SaaSで同じことができるか）の3条件を主要10業務に当てる。3つ全部Yesの業務だけが2社並列対象。「うちで人質リスクが高い業務はN件」を経営に1秒で示せます。

**2つ目、並列構成のミニマム実装を1機能だけ**。LiteLLM／LangChain／Vercel AI SDKを使い、対象1業務（例: 議事録要約）だけを「メイン: Anthropic Claude、サブ: OpenAI GPT」で並列実行できる構成にする。エラー時のフォールバックを自動化し、月数千円のコスト増で「ベンダー死亡時に1行で切替」を実現。

**3つ目、取引先監査の想定問答を1ページで作る**。「主要AIベンダー1社停止時のBCPを説明してください」への回答を、①対象業務の選定基準、②並列構成の機能範囲、③切替手順とRTOの3項目でA4 1枚にまとめる。法務・情シスで30分書けば、夏の監査で読み上げるだけで済みます。

## 3. AnthropicがPhotoshop・Blender・Fusionに直接接続──"会話駆動ワークフロー3点"

> **この章のポイント**
> - AnthropicがClaudeを Adobe Photoshop・Blender・Autodesk Fusion 等プロ向けクリエイティブツール内で直接動作させる9つの新コネクタを発表
> - 自然言語でPhotoshopレタッチ指示やBlenderのPython API経由で会話形式の3Dモデル構築・デバッグが可能に
> - AnthropicはBlender開発基金のコーポレートパトロン入り、オープンソース支援を強化

### 概要

Anthropicは、同社のAI「Claude」がAdobe Photoshop、Blender、Autodesk Fusionといったプロ向けクリエイティブツール内で**直接動作する9つの新コネクタ**を発表しました。これは単なるヘルプ機能ではなく、自然言語でPhotoshopのレタッチを指示したり、BlenderのPython APIを介して3Dモデルを会話形式で構築・デバッグしたりできる画期的なものです。さらに同社はBlender開発基金のコーポレートパトロンとなり、オープンソースへの支援を強化。プロの制作フローをAIが「内側から」支配する時代が到来しました。クリエイティブ系SaaSを業務で使っている会社は、来期に発注先・社内オペレーション・スキル要件の3つを同時に見直す時期に入っています。

### 実務で何が変わるか

煽り版の見出しはこう書かれます: 「Claudeが3DもPhotoshopも乗っ取る！クリエイター不要時代！」
実態はこう: クリエイターが消えるんじゃなく、**会話で操作できる人の時給が2倍、できない人は半分**になるだけ。差が広がる方向の話で、業界が消える話ではありません。

これ、「うちはデザイン外注してるから関係ない」で流すと、外注先の単価が会話駆動ワークフローでどんどん下がっていって、来年に「同じ予算で2倍の発注量」が当たり前になり、自社が「品質基準」を更新できないと納品物が一段陳腐化する1本です。

核心は**AI内蔵化が"プラグイン"から"標準機能"に変わった**こと。今までは「Photoshop使ってる人がChatGPTを別ウィンドウで開く」のが標準でしたが、Claudeコネクタは「Photoshopの中でClaudeに『この写真の背景を3パターン作って』と話しかけられる」段階に進化。同じことが Blender / Fusion / 主要DAW / 主要動画編集ツールでも起きるのは時間の問題です。「自社のクリエイティブ業務でAI内蔵化が効く3点」を握っておけば、外注価格・社内研修・採用要件の3つを連動して見直せます。

ただし、全クリエイティブツールにAIコネクタを入れる必要はない。会話駆動が効く"3業務"だけ先に拾うだけで、来期の予算配分は劇的に変わります。

実務で効く3つの視点。
