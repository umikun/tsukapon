## 1. Anthropic×Amazon 1,000億ドル契約──AI競争が"電力5GW"の物理戦に変わった日

この章のポイント：

- AnthropicとAmazonが10年で1,000億ドル(約15兆円)以上のインフラ契約を締結
- Claude学習・運用のために最大5ギガワット(一国分)の電力を確保
- AmazonはAnthropicに最大250億ドルを追加投資、自社チップ「Trainium」を優先採用

### 概要

Anthropicは Amazon(AWS)と、今後10年間で1,000億ドル以上を投じる歴史的規模のインフラ契約を締結した。この契約には、Claudeの学習・推論のために最大5ギガワット(小国1つ分の電力需要に匹敵)の電力を確保する条項が含まれている。AmazonはAnthropicに最大250億ドルを追加投資し、AnthropicはAmazon独自のAIチップ「Trainium」シリーズを優先的に使用することで合意した。AI企業による計算資源の"物理的独占"が決定的になり、競争軸は完全に「モデルの賢さ」から「電力・チップの長期確保量」へ移行した。

### 実務で何が変わるか

これ、「テック大企業のマネーゲームでしょ」で流すと、来月あたり上司に詰められる1本だ。

昨日のMeta×CoreWeave 210億ドルの話から、たった1日でケタが10倍に跳ね上がった。Meta 210億ドル → Anthropic×Amazon 1,000億ドル。これが示すのは、"AIは電気とチップで買う時代"が本格化したという現実だ。5GWの電力確保は、原発5基分に相当する。もはや「AIを使う」は「電力とチップの権利を押さえる」とセットの意味になった。

実務で効いてくるのは、自社が使っているAIベンダーの"電力・チップ調達力"を見抜く視点だ。ChatGPT(OpenAI) = Microsoft Azure中心、Claude(Anthropic) = AWS×Trainium、Gemini(Google) = Google自社TPU──この構図を、上司・情シス・経営層が把握していないと、"来月突然の値上げ"や"サービス縮退"に対応できない。

特に先回りで効く3つの視点がある。

1つ目、"AIのベンダー鎖国化"を前提にポートフォリオを組む。AnthropicはTrainium優先、OpenAIはAzure GPU、Googleは自社TPU──クラウド別・チップ別に分散契約しておかないと、1社のインフラ障害で業務が止まる。

2つ目、"電力コスト転嫁"を3ヶ月先まで読む。電力需要は必ず単価に跳ねる。Claude APIの料金が半年以内にプラン再編(値上げ or 低廉モデルの廃止)する確率は相当高い。予算は2026年下期に10〜20%のAIコスト上振れを織り込むのが安全だ。

3つ目、上司に"電力視点のAIレポート"を1枚作って先回り。「弊社の主要AI=Claude/ChatGPTは、それぞれAWS/Azureの電力に依存しています。5GW契約のインパクトで〜」と1枚A4で出せる人は、社内で"AIを理解している人材"の認定を一気に取れる。

## 2. トリプルアイズ"完全ローカル生成AI"──機密資料をAI化する前の実務チェックリスト

この章のポイント：

- トリプルアイズとBEXが自動車設計向け「完全ローカル生成AIシステム」を共同開発
- 外部ネットワーク非接続のオフラインRAGで、機密ナレッジを安全に活用
- 年間4,000時間の工数削減を見込む、製造業AI活用の新標準に

### 概要

トリプルアイズとBEXは、外部ネットワークに一切接続せず、機密性の高い設計ナレッジを安全に活用できる「ローカル生成AIシステム」を共同開発した。自動車業界などの製造現場では、クラウド型AIへの情報漏えいリスクが導入の最大障壁となっていたが、このシステムはRAG(検索拡張生成)技術をオフライン環境で実現。社内の膨大な技術文書・要件定義書・設計履歴から瞬時に情報を検索・要約可能だ。年間4,000時間の工数削減を見込んでおり、製造業のAI活用の新標準となる。

### 実務で何が変わるか

これ、「製造業の話でしょ」で流すと、自社の機密資料が"AI化の波"に置き去りにされる1本だ。

ここ1年、「社外秘の資料をChatGPTに貼りたいけど、情報漏洩が怖くて使えない」──この葛藤、どの会社でもあったはずだ。トリプルアイズ×BEXのシステムは、その葛藤を完全に溶かす"完全オフラインRAG"。外部ネット非接続のまま、社内の技術文書・要件定義書・設計履歴をChatGPTライクに使える構図を作った。

インパクトが大きいのは、この仕組みが製造業限定じゃないこと。法務(契約書・判例)、経理(規程・月次資料)、人事(評価基準・給与規定)、開発(設計書・ソースコード)──これらは全部"社外に出せない資料"の塊だ。クラウドAIが使えない領域で、ローカルRAGだけが唯一の正解になる。

実務で効く3つのアクション。

1つ目、"機密度マップ"を作って層別。社内資料を、①クラウドAI可(公開情報・マーケ資料)、②社外秘だが匿名化すればクラウド可、③絶対に社外に出せない(設計・契約・給与) の3層に分類。③だけをローカルRAG候補に絞る。この層別をしていないと、"全部ローカルでやる"という過剰設計(=予算超過)に陥る。

2つ目、"年間工数削減額"で稟議を通す。トリプルアイズ事例の年間4,000時間削減を時給換算(例: 4,000h × 5,000円 = 2,000万円)で上申すれば、ローカルAI構築の初期投資(1,000万前後)は半年で回収できる計算だ。稟議の通し方が一気に変わる。

3つ目、"PoC対象の資料"を3つに絞る。いきなり全社展開はNG。「FAQ回答」「技術文書検索」「規程問い合わせ」のどれかから始めれば、3ヶ月で効果測定できる。範囲を絞った人からAI導入の実績が作れるフェーズに入った。

### 社内機密資料をAI化する前のチェックリスト（3項目）

- [ ] 社内資料を①クラウドAI可 / ②匿名化すれば可 / ③絶対に社外不可 の3層で棚卸しする
- [ ] ③層のみローカルRAG候補として、年間削減工数×時給でROI試算シートを1枚作る

## 3. 神経疾患外来の"認識ギャップ予測AI"──1on1・商談の"ズレ"を先回りで止めるチェックリスト

この章のポイント：

- 患者と医師の間の「治療満足度・病状認識のズレ」を機械学習で予測
- 診察時のやり取り・問診データからギャップを事前検知、医師にアラート
- AIが"診断"から"対話の質向上"へと応用領域を拡張

### 概要

日本消化器病学会や日本臨床腫瘍学会のガイドライン改訂が報じられる中、神経疾患外来において、患者と医師の間の「治療満足度・病状に対する認識のズレ」を機械学習モデルが予測できる可能性が示された。このモデルは、診察時のやり取りや問診データから認識ギャップを事前察知し、医師にアラートを出すことでコミュニケーションの質を改善する。AIが単なる「診断」だけでなく、医療現場の「対話」を円滑にするツールとして機能し始めており、同じ仕組みはビジネスの1on1・商談・顧客対応にもそのまま応用可能だ。

### 実務で何が変わるか

これ、「医療現場の話でしょ」で流すと、来月の1on1や商談で"なんでこんなに話が噛み合わないんだ"と頭を抱えることになる1本だ。

この研究のキモは、「人と人の認識ギャップを、会話データからAIが先回りで検知する」という発想。医師・患者のコミュニケーションで起きているズレは、上司・部下、営業・顧客、先生・生徒の関係でも100%起きている。「AIが対話の質に介入できる」ことが科学的に示されたことで、1on1支援・商談支援・顧客対応支援の領域に一気に応用されていく。

実務で効くのは、"自分の会話のズレ"を可視化する習慣を今のうちに始めることだ。1on1や商談で、自分の伝えたことと相手の受け取り方に差が出ていないか。相手の満足度や懸念を、自分では気づかない形で取りこぼしていないか。これをAIに壁打ちさせる運用が、今後1年でマネージャー・営業のスタンダードになる。

実務で効く3つの視点。

1つ目、1on1・商談の直後に"認識ギャップ棚卸し"を3分だけやる。記憶が鮮明なうちに、「自分が伝えた主旨」「相手が受け取っていそうな主旨」「ズレていそうな箇所」の3点を走り書きする習慣。これだけで、次の1on1で"前回の話、どう受け取った?"と聞ける状態になる。

2つ目、"受け答えの不自然さ"をAIに壁打ちする。1on1・商談のメモをAIに貼って、「相手の不満・不安・不賛同のサイン」を3つ抽出させるプロンプトを回す。人間の脳は「自分に都合よく」メモを取るので、第三者としてのAIの指摘が一番効く。

3つ目、次回の1on1・商談の"冒頭3分の台本"をAIに作らせる。前回のメモと今回の議題から、「冒頭で確認すべき認識合わせの1問」を必ず1つ作る。これが定着すると、"話が噛み合わない会議"が月3件→月0件に減る。

### 1on1・商談の"認識ギャップ"を止めるチェックリスト（3項目）

- [ ] 1on1・商談直後に「伝えた主旨／相手の受け取り／ズレそうな箇所」の3点メモを3分で書く習慣をつける
- [ ] 月1回、そのメモをAIに壁打ちさせて"不満・不安・不賛同のサイン"を抽出する
- [ ] 次回1on1の冒頭3分の"認識合わせ質問1つ"をAIに作らせて持参する

## 4. Fortinet「Mythos」防御専用運用──社内のAI脆弱性スキャン導入チェックリスト

この章のポイント：

- Fortinetが"一般公開が危険視された"最新モデル「Mythos」を防御専用で採用
- 既存脆弱性を自律的に特定する能力、厳格管理下のみで稼働
- AIを"最終回答"ではなく"人間判断を支援する高速ツール"に再定義

### 概要

サイバーセキュリティ大手Fortinetは、Anthropicが「一般公開が危険視された」最新モデル『Mythos』を、防御専用プログラム「Project Glasswing」の一環として導入することを明らかにした。このモデルは既存の脆弱性を自律的に特定する能力に優れており、現在は厳格に管理された環境下でのみ稼働している。セキュリティアナリストの分析スピードを劇的に加速させる一方、AIモデルを"最終回答"ではなく"人間の判断を支援する高速ツール"として明確に位置付け直す動きが業界に広がっている。

### 実務で何が変わるか

これ、"IT部門の話"で流すと、3ヶ月後に社内で"Mythosで何やってますか?"と役員会で聞かれた時に答えられない1本だ。

「危険視されて一般公開できないレベルの最新AI」が、防御側で限定的に解禁された──この事実は、"AIは守りに使える"という業界コンセンサスが固まったことを示している。これまで「強すぎる攻撃AI」と「弱い防御AI」の非対称性が騒がれていたが、FortinetのGlasswingで"厳格管理下なら最強AIを防御に使える"という回路が開いた。
