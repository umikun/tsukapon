## 🔑 絶対ルール（全工程共通）

> **「Grokはデータ抽出、Claudeは分析と下書き、投稿ボタンは必ず人間が押す」**

| ✅ AIに任せる | ❌ AIに絶対させない |
|---|---|
| Grok: X全体の伸びた投稿の抽出（数値・URL） | 投稿ボタンの自動クリック |
| Claude: バズ理由の言語化・伸び要因の深掘り | リプの自動送信 |
| Claude: 引用RT・リプの下書き生成（my-clone口調） | いいね／フォローの自動実行 |
| Claude: 自分の過去投稿との重複チェック | 同一テキストの大量連投 |
| Claude: 週次データの集計・KPI推移分析 | 複数アカウント連動運用 |

→ これを守れば①〜④は全てX規約の範囲内で運用可能。

## 🧭 ①〜④ ステップ詳細（番号の定義）

日次フロー内の「① 実施」〜「④ 実施」は、本フロー独自の番号体系（①から振り直し）。元ネタの [[note-20260426間違い]] からは**規約適合範囲のみを抜粋**しており、kuso_inc番号の⑥⑦⑨に相当する「自動リプ／自動投稿／複数垢連動」はX規約違反のため**本フローでは一切実施しない**。

### ① Claude Cowork を起動する（= 画面共有型のAI指揮官を立ち上げる）

**何をする工程か**
- macOS用デスクトップアプリ **Claude Cowork** を起動し、**PC画面のスクショ読取・ローカル操作（Computer Use）** をClaudeに許可する状態を作る。
- これにより、この後 ② で開く Chrome の X.com 画面を Claude が「視覚的に読み」ながら、同時に Obsidian（このvault）側に下書き・分析結果を書き出せるようになる。
- 言い換えると、① は **"Claude を操縦席に座らせるだけ"** の準備工程。まだ投稿もスクレイピングもしない。

**具体的な操作**
1. macOSのアプリケーションから **Claude Cowork** を起動(未インストールなら [[SNS運用/archive/note/note-20260324-computer-use.md]] の手順で導入)
2. 画面収録・アクセシビリティ権限を付与（初回のみ）
3. 「Claude Code / Claudian」ではなく、**Cowork側のチャット欄**を開く（vault横断操作は Claudian、画面操作は Cowork、の役割分担）
4. セッションが立ち上がったら、後続の ②（Chrome起動）→ ③（TL分析プロンプト投下）へ進む

**この工程で"しない"こと**
- ❌ 投稿ボタンを押させる設定
- ❌ いいね／フォローの自動化
- ❌ 複数Xアカウントの同時ログイン
- ✅ 画面を"見る"・vaultに"書く"・Chromeを"操作する"ところまで

**所要時間**: 約5分（起動＋権限確認のみ）

**紐づくKPI**: 直接のKPI寄与は無し（②③④の土台工程）

### ② Claude in Chrome で x.com を開く

Chrome拡張のMCP経由で、**自分のログイン済みセッション**のまま X.com を開く。Claudeはこのタブの内容を読み取れるようになる（= TL巡回・下書き貼り付けの準備）。この時点で Grok タブも併せて開いておくとスムーズ（③A-1で使う）。

### ③ TL分析＋下書き生成（③A＋③B／ハイブリッド運用）⭐ UPDATED 2026-04-23

**③A データ抽出＋伸び要因分析（ハイブリッド: Grok → Claude）**

| サブ工程 | 担当 | 処理内容 |
|---|---|---|
| **③A-1** | **Grok（有料プラン）** | 過去3時間のAI関連投稿でインプ5000超のものを **10件抽出**（URL／投稿者／インプ数／冒頭1文） |
| **③A-2** | **Claude in Chrome** | ③A-1で取得した10件のリストを渡し、伸び要因を「フック／具体性／関係性トリガー／損失回避」の4観点で1行ずつ分析。伸びたフレーズ・型を3つ列挙 |

**ポイント**
- Grokはデータ取得だけ。**分析・下書き作成はGrokに振らない**（my-clone口調・7つの型・NGワード等の独自ルールが効かないため）
- ③A-1でGrokから返ったリストは Claude in Chrome のプロンプト冒頭に**そのまま貼り付ける**
- Grokが使えない日は、フォールバックとして Claude in Chrome のTLスクロールで代替可（精度は落ちる）

**③B 下書き生成（Claude 一気通貫）**

- 10件のうち「AI × 業務効率化 × 初心者向け」ペルソナに合う3件を選定
- その投稿への引用RT下書きを生成 → `SNS運用/post/draft/YYYYMMDD_quote_draft.md` に保存
- 下書き生成後は **Claudian の `/quote-rewrite`** で my-clone口調＋関係性キーワード（上司／会社／同僚／部下／取引先の分散）＋7つの型判定＋140字チェック＋安全運用チェックリストを付けた最終版に仕上げる

### ④ 下書きの選別＋手動投稿

人間が下書きを読み、**自分の言葉で1回書き直してから手動で投稿ボタンを押す**。1日3本以内。kuso_inc番号⑥⑦⑨（自動リプ／自動投稿／複数垢）は**実施しない**。

- [ ] Claude/Grokに**投稿ボタンを押させていない**か？
- [ ] 複数アカウントを運用していないか？（メイン1垢固定）
- [ ] Grokの抽出リストを**コピペだけ**で流し、**Grokに下書き生成させていない**か？（口調・型の再現性確保）

→ 1つでも ❌ があれば、その投稿はキャンセル。

- 精度は落ちるが、運用自体は止めずに継続できる
- 「Grok枯渇 → Claudeに任せる」は可。「Claude枯渇 → Grokに下書きまで任せる」は**不可**（品質崩壊）
